第22章 变量命名的洁癖
逻辑错误是能力问题。命名错误是态度问题。
CHEP后第十八天。
古法编程班第三天。
903室多了一张长桌。是老李从数据中心的会议室搬来的，六张折叠椅拼在两边。白板前面留出了一块站立区域——马农教课的地方。服务器机架被推到了房间的后半部分，中间用一道临时隔板分开。
前面是教室。后面是机房。
声音隔不住。十五台服务器的风扇声是永恒的背景白噪音。但没人介意。在CHEP后的世界里，这种声音意味着文明还在运转。
今天的内容是方法（函数）的参数传递和返回值。
马农在白板上写了一段代码：
``java
static int add(int a, int b) {
    return a + b;
}
`
"上次讲了方法的定义和调用。今天讲返回值。"他指着int。"这里的int不是参数的类型——是返回值的类型。这个方法做完事之后，会把一个整数交回来。"
"交回给谁？"产品经理问。
"谁调用它，就交回给谁。"马农写了调用：
`java
int result = add(3, 5);
System.out.println(result);  // 8
`
"add(3, 5)做了计算，把8交回来。result接住它。"
他放下笔。"现在——练习。"
白板上出现了三道题：
`
练习1：写一个方法，输入半径r，返回圆的面积（πrr）
练习2：写一个方法，输入一个整数数组，返回最大值
练习3：写一个方法，输入一个字符串，返回反转后的字符串
`
"第二第三题需要用到数组和String。"马农说。"不会的看白板左边——昨天的笔记还在。二十分钟。"
十八分钟后。
大部分人完成了练习1和练习2。练习3有一半人还在挣扎——反转字符串需要理解charAt()和循环拼接，对零基础的人来说是一个小跳跃。
马农开始巡视。
他不是那种走来走去嘘寒问暖的老师。他只是站在一个人身后看三秒，然后走到下一个人身后。不说话。看完就走。
直到他走到大四实习生面前。
实习生叫赵天。简历上写"精通AI辅助开发"，实际上意思是"精通给Origin写提示词"。CHEP之后，他的全部技能变成了一张废纸。他来903室是因为没别的地方可去。
马农看了他的屏幕三秒。
然后他的手停了。
赵天的代码是这样的：方法名s，变量名x是3.14，y是结果。技术上说，这段代码是对的——输入半径返回面积，能编译能运行结果正确。
马农的声音很平："这是什么。"
赵天回头看他："练习1啊。圆的面积。"
"我是问——s是什么。x是什么。y是什么。"
赵天愣了一下："s是方法名……x是π……y是结果。"
"你自己知道。三天后你还知道吗？三个月后呢？如果另一个人来看这段代码，他怎么知道s是计算面积？"
赵天有点心虚："加注释？"
马农没有提高音量。但整个903室都安静了——他的语速变慢了，在克制。
"注释是给'为什么'用的。不是给'是什么'用的。如果你的变量名需要注释来解释'它是什么'——那变量名就是错的。"
他走到白板前，写了两版代码做对比：错误示范是s(double r) + x + y，正确示范是calculateArea(double radius) + pi + area。
"同样的逻辑。同样的结果。但第二个版本，任何人看一眼就知道这个方法在做什么。"
他转过身。
"变量命名有三条规则。"
他在白板上写：
`
1. 名字必须自解释：看名字就知道它是什么
2. 禁止单字母变量：a/b/c/x/y/z——除了循环变量i/j/k
3. 方法名必须是动词：calculate、get、print、check
`
"这三条不是建议。是规矩。"
他看着十二个人。
"从今天开始，我检查你们代码的时候，只要看到a、b、temp、data这种名字，整段代码重写。不管逻辑对不对。"
有人不服。
是那个写过Python的程序员——叫周明。三十岁出头，之前在一家创业公司做后端。CHEP前他用Origin写代码已经两年了，在那之前他确实手写过Python。
"我觉得这也太严了。"周明说。"能跑就行吧？反正现在是应急状态……"
马农看着他。
"你写过Python。"
"对。"
"你维护过别人的Python代码吗？"
周明想了想："维护过。接了一个前同事的项目。"
"你看他代码的第一感受是什么？"
周明沉默了两秒。然后苦笑了一下。
"想杀人。"
903室里几个人笑了。
"为什么想杀人？"马农追问。
"因为看不懂。变量名全是a1、a2、tmp、res。方法名叫do_it、run、process。注释写的是中文但跟代码完全对不上——"
"够了。"马农打断他。"你刚才描述的每一个问题，都始于同一个原因：命名。"
他走回白板。
"代码被阅读的次数远远多于被编写的次数。你写一次，可能被读一百次——被你自己、被同事、被三年后接手项目的陌生人。如果你的命名让人看不懂，你就是在对这一百个人犯罪。"
他的声音仍然很平。但"犯罪"两个字让所有人都坐直了。
午饭后。下午的课是综合练习。
马农在白板上写了一个作业：
`
综合练习：写一个成绩管理程序
- 存储5个学生的名字和成绩
- 方法1：打印所有学生的名字和成绩
- 方法2：输入一个名字，返回对应的成绩
- 方法3：返回最高分的学生名字
`
"用你们学过的东西——变量、数组、循环、条件、方法。四十分钟。"
这道题比前两天的难度跳了一级。不是单一概念的练习，而是需要把多个概念组合在一起。
教室里安静了。键盘声稀疏了——思考的时间比敲键盘的时间多。
马农坐在角落里。他不催。好的教学从来不在于教了多少，而在于留了多少空间让学员自己想。
二十分钟后，刘韧第一个完成了。代码整洁，命名规范。马农看了一遍，没说话。这是他的最高评价之一——不说话意味着没问题。
三十分钟后，林夕也完成了。
她的代码是这样的：
`java
public class GradeManager {
    static String[] studentNames = {"林夕", "刘韧", "赵天", "周明", "张伟"};
    static int[] studentScores = {85, 92, 76, 88, 70};
    static void printAllStudents() {
        for (int i = 0; i < studentNames.length; i++) {
            System.out.println(studentNames[i] + ": " + studentScores[i]);
        }
    }
    static int findScoreByName(String targetName) {
        for (int i = 0; i < studentNames.length; i++) {
            if (studentNames[i].equals(targetName)) {
                return studentScores[i];
            }
        }
        return -1;  // 没找到
    }
    static String findTopStudent() {
        int maxScore = studentScores[0];
        int maxIndex = 0;
        for (int i = 1; i < studentScores.length; i++) {
            if (studentScores[i] > maxScore) {
                maxScore = studentScores[i];
                maxIndex = i;
            }
        }
        return studentNames[maxIndex];
    }
}
`
马农看了她的代码。
从上到下。每一行。
他注意到了几个细节：studentNames而不是names、studentScores而不是scores——带上了上下文。findScoreByName而不是getScore——方法名描述了行为。targetName而不是name——区分了参数和字段。
"你听进去了。"马农说。
林夕抬头看他。
马农又看了一遍第二个方法。
"return -1。"他指着那行。"为什么返回-1？"
"没找到的时候，返回一个不可能是成绩的值。表示'没有结果'。"
"如果成绩可以是负数呢？"
林夕愣了一下："……那-1就不行了。"
"所以怎么办？"
林夕想了几秒："用一个特殊的类型？比如……可以为空的类型？"
马农没有直接回答。他在白板角落写了一个词：
`
Optional<Integer>
`
"第二周的内容。先记住这个问题。"
林夕点头。她在笔记本上写了下来——第23页。前22页已经写满了。
四十分钟到了。
赵天的代码还没写完。他卡在第三个方法上——找最高分。
但让马农真正停下来的不是他没完成。是他已经完成的两个方法的命名。
`java
static void print() {
    for (int i = 0; i < n.length; i++) {
        System.out.println(n[i] + ": " + s[i]);
    }
}
static int get(String a) {
    for (int i = 0; i < n.length; i++) {
        if (n[i].equals(a)) {
            return s[i];
        }
    }
    return -1;
}
`
n。s。a。print。get。
马农站在赵天身后。
今天上午他花了二十分钟在白板上写了正确示范。他列了三条规则。他说了"只要看到a、b、temp这种名字，整段代码重写"。
赵天转过头。看到马农的表情。
严格来说，马农没什么表情。他的脸一直是那样——平静的、不带情绪的。但赵天后来跟别人描述这一刻的时候说："他的眼神变了。不是愤怒。是失望。失望比愤怒可怕。"
"重写。"
一个词。
"啊？"
"整段。重写。"马农的声音没有抬高。"把n改成studentNames。把s改成studentScores。把a改成targetName。把print改成printAllStudents。把get改成findScoreByName。"
"可是……功能是对的啊——"
"我不关心功能。"
903室彻底安静了。连风扇声都像是屏住了呼吸。
"功能对是最低要求。及格线。连讨论的资格都不算。"马农说。"你写的代码不是给编译器看的——编译器什么都认。你写的代码是给人看的。你用n和s，你自己今天知道是什么意思。明天呢？下周呢？别人呢？"
他的语速仍然很慢。每个字都像是在键盘上按下去的——确定的，不可撤回的。
"代码不是你一个人的事。"
他停顿了一秒。
"从今天开始，每个人交作业之前，先检查变量名。如果我看到一个没有自解释性的变量名——不管你的逻辑多完美——退回重写。"
赵天的脸红了。不是羞愧——是一种更复杂的东西。他后来在笔记本的第一页写了一句话：马农从来不骂人。但他说"我不关心功能"的时候，比骂人狠一百倍。
那天傍晚。课结束了。
赵天重写了代码。第二版的命名明显好了——studentNames、studentScores、findTopStudent。他还主动把注释改了，从"// 遍历数组"改成了"// 如果找不到目标学生，返回-1表示查询失败"。
马农看了第二版。
"可以。"
两个字。但赵天像是得了一枚勋章。
林夕走的时候。
她注意到白板右下角多了一行手写的字——不是今天上课写的，是马农后来加的。
字很小，不仔细看发现不了：
`
代码写出来是给人看的，顺便给机器执行。
                    —— Harold Abelson
`
她记住了这句话。后来它成了她教新人时说的第一句话。
那天晚上。903室。
马农在培训计划上画了第三天的勾。然后在旁边加了一行批注：
`
Day 3: 命名规范。
赵天的问题不是他笨。是习惯。
习惯比能力更难改。
但必须改。不改的人写不出能活过一年的代码。
`
他合上了本子。
角落里那个人事评估文件还开着。他在林夕名字后面加了两个字：
`
林夕 — 零基础但理解力强。问问题的方式对。命名有意识。
``
"命名有意识"——在马农的评价体系里，这四个字的权重比"算法能力强"重得多。
因为算法可以教。意识教不了。
窗外。深圳的第十八个夜晚。
903室的灯亮着。白板上的代码在灯光下像一面不会说谎的镜子——写什么就是什么，没有PS、没有滤镜、没有Origin替你补全。
马农关了培训计划的文件。打开了另一个文件——903室的系统监控。
用户数：4,217。消息数：38,000+/天。峰值QPS：680。CPU使用率：31%。
他上周优化的CAS缓存和请求合并还在稳定运行。四千多人在用他写的系统找家人、传消息、确认平安。
但他知道——这个系统是他一个人维护的。如果他倒了，没有人能接手。不是因为代码难——而是因为没有人理解代码。
所以他必须教。
不是教怎么写for循环。是教怎么写让别人能看懂的for循环。
不是教怎么用变量。是教怎么给变量起一个三年后还能看懂的名字。
不是教技术。是教态度。
马农关了灯。
十五台服务器的LED灯在黑暗中闪烁。
明天教类和对象。
