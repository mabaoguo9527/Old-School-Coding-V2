第21章 古法编程班

所有伟大的技艺，都从最笨拙的第一步开始。

CHEP后第十六天。

903室的白板被擦干净了。

不是那种匆忙的擦——残留着前一天架构图的影子、被汗手蹭花的墨迹——而是用湿毛巾仔仔细细擦了三遍，连边角的笔迹都没留。

马农在白板正中央写了一行字：

``
古法编程·第一课
`

字很小。但很工整。不像他平时在白板上画架构图时那种狂草——那些线条是给自己看的，可以潦草。这一行是给别人看的。

他在白板下方又写了一行：

`
你不需要Origin。你需要理解。
`

八点整。

来了十二个人。

不是马农要求的二十个——CTO答应的"能学的人"还在路上。但903室的消息系统在过去两天传遍了一条消息："数据中心B栋903室，有人在教手写代码。"

十二个人里，六个是腾讯的在职员工——三个提示词工程师、一个前端、一个测试、一个产品经理。剩下六个是数据中心里被困的外部人员——两个外包运维、一个华为的驻场工程师、两个在附近上班走不了的程序员（自称"写过Python"）、还有一个大四实习生，简历上写着"精通AI辅助开发"。

他们找到903室的方式各不相同：有人是在消息板上看到的，有人是听老李提到的，有人只是路过看到门开着。

但他们来的原因只有一个。

林夕坐在最前面。

她是唯一一个带了两样东西来的——一台装好了JDK 17的笔记本电脑，和一个写满了笔记的本子。本子的第一页写着：int a = 1; 旁边画了一个箭头，指向一个方框，方框里写着"1"。

这是她两天前学的。

马农站在白板前。扫了一眼。

他没有自我介绍。没有问"大家都是什么背景"。没有说"欢迎来到古法编程班"。

他说："打开终端。"

十二个人面面相觑。有人掏出笔记本电脑，有人还在翻包。那个产品经理犹豫了一下："我没有开发环境……"

"有JDK的举手。"

三只手。林夕、一个华为驻场、一个自称写过Python的程序员。

马农从抽屉里拿出一个U盘。和昨天给林夕的不一样——这个更大，32GB，贴着一个手写的标签："JDK17-offline-full"。

"传一圈。装好了喊我。"

他把U盘递给最近的人，然后转身回到白板前。

等待的时间里，他在白板上写了三行：

`
变量：给一个值起名字
循环：重复做一件事
条件：做选择
`

就这三行。没有更多。

"这就是今天的全部内容。"马农说。"你们如果能真正理解这三行字，就比Origin强。"

有人笑了。觉得他在开玩笑。

他没有开玩笑。

十五分钟后。十二台笔记本电脑都装好了JDK。

"打开命令行。"马农说。"输入：javac -version。"

十二道键盘声。然后是此起彼落的声音：

"javac 17.0.2。"
"我的也是。"
"成功了。"

"好。"马农在白板上写下第一段代码——一个最基础的Java程序：public class Hello，main方法里只有一行System.out.println("hello")。

"抄。一个字都不要错。保存为Hello.java。然后在命令行输入javac Hello.java。再输入java Hello。"

十二个人开始抄。

三分钟后。

"报错了。"那个产品经理说。"说什么class……找不到。"

"文件名。"马农没回头。

"啊？"

"文件名必须和class名一致。Hello.java，H大写。"

"哦……"

又过了一分钟。

"成功了！"产品经理的声音里有一种不成比例的兴奋。屏幕上打印了一个单词：hello。

"这就……跑起来了？"

"这就跑起来了。"马农说。

旁边有人嘀咕："这也太简单了吧……"

马农听见了。他没有反驳。

他在白板上擦掉了Hello程序，写了一段for循环：for (int i = 0; i < 10; i++)。起点、条件、步进，三件事一行写完。机器按照这三个条件，重复执行大括号里的内容。

他停顿了一秒。

"抄。运行。看输出。"

一分钟后。

"0到9？为什么不是1到10？"那个大四实习生问。

马农看了他一眼。

"因为计算机从零开始数。"他说。"这不是设计缺陷。这是逻辑——数组的第一个元素在内存的第零个偏移量。从零开始，是因为机器不撒谎。"

实习生点了点头，似懂非懂。

林夕在旁边轻声说："我昨天问过马农一样的问题。"

"你怎么理解的？"实习生问。

"你就想象一把尺子。"林夕说。"零刻度线不是'没有'，是'起点'。"

马农没看她。但他手里的笔停了一秒。

接下来的两个小时，马农教了三样东西。

第一：变量。int age = 25——整数。String name = "林夕"——文字。double pi = 3.14——小数。他一句一个定义，没有多余的字。

"为什么叫double不叫decimal？"有人问。

"因为它用64位存储，是float的两倍。名字来自精度，不是来自含义。"

"那float呢？"

"32位。精度不够。用double。"

三句话。没了。

第二：条件。if (age >= 18)——如果成年，else——否则未成年。"这是编程里最接近人类思维的结构。你做每一个决定，本质上都是一个if-else。"

"那if里面可以再套if吗？"林夕问。

"可以。但不要超过三层。"

"为什么？"

"因为你自己会看不懂。"

有几个人笑了。马农没笑。他是认真的。

第三：循环的变体。while (count < 5)——while循环是"我不知道要做几次，但我知道什么时候停"。do-while是"不管条件如何，至少做一次"。

他放下笔。

"这三种循环本质上是同一件事：重复。区别只在于你什么时候决定停下来。"

中场休息。

903室外面的走廊里，十二个人三三两两地站着。有人拿着矿泉水发呆，有人在手机上反复看自己写的代码——虽然手机上没法编译，但他们在看。

产品经理走到林夕旁边："你之前就跟马农学过？"

"两天。"林夕说。"只学了变量和赋值。"

"他一直这么教的？就这么……直接？"

"他说话从来不超过二十个字。"

"那你怎么学的？"

林夕想了想。"他写在白板上的东西，我会抄三遍。不是抄代码，是抄逻辑。第一遍抄字面意思——这行代码做了什么。第二遍抄为什么——为什么要这样写。第三遍抄如果不这样写会怎样。"

产品经理有点吃惊："这是他教你的方法？"

"不是。是我自己想的。"林夕笑了一下。"他只给答案，从来不给方法。"

走廊尽头，马农站在窗边。他不是在休息。他在看一张手写的清单——培训计划v1.0。第一周的内容他提前做了标注：

`
第一天：变量、循环、条件        ← 今天
第二天：方法（函数）
第三天：数组
第四天：类与对象（概念）
第五天：类与对象（实践）
第六天：异常处理
第七天：综合练习——写一个计算器
`

他用马克笔在"第一天"后面画了一个勾。

然后他看了一眼走廊里的十二个人。

十二个。

附录C写的是"广东省1-3人"。那是指CHEP前就会手写代码的人。

现在他要把这个数字变成十五、二十、五十。

时间不够。但开始了。

下午。第二节课。

马农教了一样新东西：方法。

他把上午的for循环改成了方法——static void printNumbers(int start, int end)，把重复的逻辑包起来，需要的时候叫名字就行。"方法就是给一段逻辑起个名字。"

"像函数？"一个写过Python的程序员问。

"一样的东西。"马农说。"Java管它叫方法。"

"那static是什么意思？"

马农停了一秒。这个问题的完整回答涉及类的实例化、内存分区、方法调用栈。

"现在不用管。"他说。"先加上。第四天讲。"

这是马农教学的一个原则——不在这一层解释下一层的东西。他在培训计划里写过一行注释：每天只教一层。学员消化不了的概念延后，绝不提前。

"好。接下来——"马农在白板上写了一个新练习：

`
练习1：写一个方法，输入一个整数n，打印从1到n。
练习2：写一个方法，输入一个整数n，打印从n到1（倒数）。
练习3：写一个方法，输入两个整数a和b，打印它们之间的所有偶数。
`

"写。二十分钟。"

903室安静了下来。只有键盘声。

马农在白板旁边的折叠椅上坐下。他没有巡视、没有走来走去看学员的屏幕。他在等。

七分钟后，第一个完成的人举了手。是华为的驻场工程师——一个三十出头的男人，名字叫刘韧。

"做完了。三个都对。"

马农走过去了一眼——一个for循环从1打印到n，循环边界处理得干净利落。

"你以前写过Java。"马农说。不是问句。

"写过一点。大学的时候。"刘韧有点不好意思。"后来就全用Origin了。"

"手感还在。"马农说。

刘韧愣了一下。他意识到这是一句肯定。

十五分钟后，大部分人完成了。

但有三个人还在挣扎。产品经理、大四实习生、和一个外包运维。

产品经理的代码里用了i / 2 == 0来判断偶数。马农看了三秒。

"取余。不是除。"

"什么？"

"i % 2 == 0。百分号。不是斜杠。/是除法，%是取余数——除以2余数为零，就是偶数。"

产品经理恍然："哦！我混了……"

"正常。"马农说。这可能是他说过的最温和的两个字。

旁边的大四实习生更惨。他的代码连编译都没通过——少了一个大括号。他盯着报错信息，红着脸。

马农没有直接告诉他哪里错了。

"报错信息第一行。"马农说。"它告诉你行号了。"

实习生看了看：Loop2.java:8: error: ';' expected。

"第8行……"他翻了翻自己的代码。"啊！这里少了个大括号。"

"编译器不会骗你。"马农说。"它比你更诚实。报错信息是你最好的老师——比任何人都准确。"

他停了一下。

"包括我。"

第一天的课在下午四点结束了。

马农在白板上写了家庭作业：

`
今日作业（明早八点前完成）：
1. 写一个方法，输入n，计算1+2+3+...+n的和
2. 写一个方法，输入n，判断是否是质数
3. 思考题：如果n是负数，你的方法会怎样？
`

"质数……怎么判断？"有人问。

"想。"马农说。"质数的定义是什么，你就怎么判断。编程不是记忆。是把逻辑翻译成代码。"

十二个人陆续离开了。走的时候，903室比来的时候安静得多。没有人闲聊。每个人都在想那三道题。

最后走的是林夕。

她在门口停了一下。

"马农。"

"嗯。"

"你为什么教我们for循环？"

马农抬头看她。

"你不是说过——你需要能写代码的帮手。for循环……离写一个服务还很远吧？"

马农把马克笔放下。

"你见过建房子吗？"他问。

"见过。"

"地基打多长时间？"

林夕想了想："好几个月？"

"嗯。上面盖楼可能只要半年。但地基要打三个月。"马农说。"for循环就是地基里最下面那层钢筋。你不理解循环，就不理解数组。不理解数组，就不理解数据结构。不理解数据结构，就不理解算法。不理解算法——"

他停了。

"——就不理解代码在做什么。"林夕接上了。

马农看了她一秒。

"回去写作业。"

林夕走了。

那天晚上。903室。

马农一个人坐在终端前。屏幕上不是代码——是一份名单。

十二个人。他给每个人写了一行评估：

`
刘韧 — 有基础，手感在。可以加速。
林夕 — 零基础但理解力强。问问题的方式对。
张伟（外包运维）— 有Linux经验，命令行熟练。缺编程思维。
陈明（测试）— 逻辑严谨但动手慢。适合质量方向。
......
`

十二行。每人一句话。

他师父当年也是这么做的。每次教完一课，在本子上写一行评语。马农十七岁那年偷看过师父的本子，在自己名字后面看到四个字：

"笨。但勤。"

马农当时不高兴了好几天。后来才明白——"勤"是师父给过的最高评价。

他在名单最下面加了一行：

`
总评：12人。可造之材3-4人。其余需要更多时间。
时间是最大的敌人。
`

他保存了文件。

窗外的深圳和十六天前没什么不同——黑的。但903室里有十五台服务器的灯光，有白板上还没擦掉的for循环，有十二个人留下的键盘温度。

马农打开了培训计划，把"第一天"的勾描深了一点。

然后在旁边写了一行小字：

`
第一天。十二个人。够了。
``

明天教方法的参数传递和返回值。

后天教数组。

时间不够。但开始了。
