第25章 第一个门徒
代码运行起来的那一刻，不是机器在回应你。是世界在回应你。
CHEP后第二十八天。
古法编程班第十三天。第二周结束。
马农在白板上写了第二周的毕业题：
``
综合项目：903室物资管理系统
要求：
1. 设计至少三个类：Item（物资）、Warehouse（仓库）、Manager（管理器）
2. 物资属性：名称、数量、类别、入库日期
3. 仓库功能：入库、出库、按类别查询、库存预警（数量<10时提醒）
4. 管理器功能：管理多个仓库、生成库存报告
5. 从命令行交互——输入指令操作
不允许使用任何外部库。只用JDK标准库。
时间：半天。
`
这道题是真实需求。903室的物资管理目前全靠老李手写在纸上——柴油存量、服务器零件、食物饮水、工具设备。纸质记录的问题是查询效率低、容易出错、无法自动预警。
马农出这道题不是为了练习。是为了让学员写一个能实际部署的系统。
上午。
十一个人开始写。
这是第一次"半天级"的项目。之前的练习最长不超过一小时。半天——意味着他们需要自己规划时间、自己设计类结构、自己决定方法的参数和返回值。
没有人问问题。
不是因为不敢问——是因为马农在第一周就立了一个规矩：写代码的时候不准问。先自己想。想不通的记下来。写完之后统一问。
"写代码的过程就是思考的过程。"马农说过。"你一卡住就问，脑子就停了。停了就长不出自己的思路。"
两个小时后。
刘韧第一个完成。代码四百多行。三个类层次清晰——Item是纯数据载体，Warehouse封装了所有存储逻辑，Manager负责多仓库协调。
马农看了十分钟。逐行。
"你的Warehouse里用了ArrayList<Item>。"马农说。"按名称查找的时候是遍历——O(n)。如果物资种类超过一千种？"
"加索引？用HashMap<String, Item>做名称映射？"
"可以。考虑一下什么时候用List、什么时候用Map。不是都用List就对的。"
刘韧点头。他在代码旁边加了一行注释：// TODO: 大量物资时考虑HashMap索引。
马农看到了那行注释。
"好习惯。"他说。"TODO是给未来的自己留的路标。"
三个小时后。
大部分人完成了主体功能。但只有四个人实现了"命令行交互"——从键盘输入指令操作系统。其他人的程序是"写死数据跑一遍"。
马农没有批评。命令行交互需要解析字符串输入、处理非法输入、循环等待——对第二周的学员来说已经超纲了。能实现核心逻辑就够了。
他一个一个看。
看到赵天的时候，停了一下。
赵天的代码有三百行。不是最长的，也不是最短的。但有一个细节——他的Item类覆写了toString()方法，而且格式和林夕之前写的Server类的toString()非常像。
"你参考了林夕的代码。"马农说。
赵天的脸红了："我……看了一眼她上次的写法。"
"不是坏事。"马农说。"好的代码应该被学习。但你要标注出处。在注释里写'参考林夕的Server.toString()设计'。"
"啊？写在代码注释里？"
"代码是有作者的。"马农说。"你用了别人的思路，就要承认。这不是学术规范——这是尊重。"
赵天愣了一下。然后他真的在代码里加了一行注释：
`java
// toString格式参考了林夕的Server类设计（Day 9）
`
四个小时。
林夕的代码还没写完。
这让马农有一点意外。她通常是前三个完成的。
他走过去看了一眼她的屏幕。
林夕不是在写核心逻辑——核心逻辑她两小时前就写完了。她在写一个马农没有要求的功能：库存变动日志。
`java
public class InventoryLog {
    private final List<String> logs = new ArrayList<>();
    public void record(String action, Item item, int quantity) {
        String timestamp = LocalDateTime.now()
            .format(DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm"));
        String logEntry = String.format("[%s] %s: %s x%d",
            timestamp, action, item.getName(), quantity);
        logs.add(logEntry);
    }
    public void printRecentLogs(int count) {
        int start = Math.max(0, logs.size() - count);
        for (int i = start; i < logs.size(); i++) {
            System.out.println(logs.get(i));
        }
    }
    public void saveToFile(String filename) throws IOException {
        try (var writer = new BufferedWriter(new FileWriter(filename))) {
            for (String log : logs) {
                writer.write(log);
                writer.newLine();
            }
        }
    }
}
`
她额外加了一个类。四个方法。还加了文件持久化——把日志写到磁盘上。
"我没有要求这个。"马农说。
林夕抬头："我知道。但我想……如果老李真的要用这个系统，他肯定想知道谁在什么时候拿了什么东西。纸上记录能查，电子系统也应该能查。"
"你怎么想到saveToFile的？"
"我想到如果服务器重启了，内存里的日志就没了。所以要存到磁盘上。"
马农看着她的代码。看了很久。
LocalDateTime、DateTimeFormatter、BufferedWriter、try-with-resources——这些API她不是课上学的。是她自己翻JDK文档找到的。
他注意到代码里有几处小问题：FileWriter没有指定编码（默认编码在不同系统上可能不同）、saveToFile每次都是覆盖写入（应该追加）、没有并发保护（如果两个人同时操作仓库呢）。
但这些都是第三周的内容。
她现在能做到这一步——在第二周结束的时候，自己想到了需求以外的功能，自己查了API，自己写了四个方法和文件IO——
"多长时间了？"马农问。
"两个小时。核心逻辑两小时。日志功能……又花了两个小时。"
"你原来的核心逻辑还在？"
"在。我先写了一版不带日志的，确认能跑。然后拷了一份，在拷贝上加日志功能。"
"为什么拷贝？"
"怕改坏了原来能跑的版本。"林夕说。"你第一周说过——能跑的代码不要轻易碰。先备份。"
马农的手指停了一秒。
他记得。他确实说过这句话。在第三天，改DNS服务的时候随口说的。他以为没人会记住这种顺口的话。
"写完了吗？"他问。
"差不多。还有一个功能想加——库存预警发送到消息系统。就是把预警信息通过903的消息板推出去。"
"那需要Socket。"
"我知道。"林夕说。"所以我先留了一个接口。等第三周学了网络编程再实现。"
她指着代码里的一行：
`java
// TODO: 第三周学完Socket后，实现预警消息推送
// 接口预留：public void sendAlert(String message)
`
马农看着这行注释。
他没有立刻说话。
他走回了白板前。站了三秒。然后转身面对所有人。
"过来。"他对林夕说。"把你的代码投屏。"
903室没有投影仪。"投屏"的意思是——把笔记本电脑放在白板前面的桌子上，所有人围过来看。
林夕紧张了一下。她不确定马农要做什么。是要批评她自作主张写了多余功能？还是——
"给大家讲一下你的代码。"马农说。
林夕愣住了："我……讲？"
"你写的。你讲。从第一行开始。"
林夕站在白板前。十个人围在她的笔记本旁边。
她的声音一开始有点发抖。
"这个……这个是Item类。物资。属性有名称、数量、类别、入库日期。"
她翻到Warehouse类。
"仓库用ArrayList存物资。入库的时候先检查有没有同名物资——有的话加数量，没有的话新建。出库的时候检查库存够不够。"
讲到InventoryLog的时候，她的声音稳了一些。
"这个不是作业要求的。但我觉得……一个真的在用的系统，应该有日志。不然出了问题你不知道什么时候出的。"
"saveToFile是存盘。用BufferedWriter写到文件里。这样即使程序关了，日志还在。"
她指着最后那个TODO注释：
"这里是预留接口。现在还不能实现。但我先留了位置。等学了网络编程再补。"
她讲完了。站在那里。手指不自觉地绞着马尾辫的尾巴。
903室安静了三秒。
马农看着她。
"你做了三件事。"他说。
"第一——你完成了基础要求。核心逻辑对，命名规范，结构清晰。这是及格线。"
"第二——你自己想到了需求以外的功能，而且这个功能是实际有用的。这说明你在'理解需求'，不是在'完成任务'。"
"第三——你留了一个接口给未来。一个你现在不会写的功能，你没有硬写——你留了位置。这说明你知道自己的边界在哪里，但你没有因为不会就放弃设计。"
他停了一下。
"这三件事加在一起，叫做——"
他在白板上写了两个字：
`
工程思维
`
"不是写代码。是做工程。区别在于——写代码是解决当下的问题。做工程是解决当下的问题，同时为将来的问题留出空间。"
他转向林夕。
十个人看着他们两个。
马农说了一个词。
"可以。"
两个字。
但在马农的评价体系里，这两个字的重量，大概等于别人说"太出色了"、"超出预期"、"我为你骄傲"。
林夕的眼睛湿了。她没有说话。只是点了一下头。
然后她轻声说："谢谢。"
马农已经转回了白板。他在写第三周的课程表。
他没有回头。
那天下午。下课后。
刘韧在走廊里找到林夕。
"恭喜。"他说。
"恭喜什么？"
"马农说'可以'。我跟了他两周，他对我说的最多的是'嗯'。"
林夕笑了一下。然后她正了正脸色。
"刘韧……你觉得马农为什么让我讲代码？"
刘韧想了想："因为你的代码好？"
"不只是。"林夕说。"他在给我一个机会——站在前面讲。他在试我能不能教别人。"
刘韧看着她，有点惊讶。
"他写的培训计划我看到过。"林夕说。"第四周的最后一天，有一行：'选一名学员做助教，协助第二批培训'。"
"你偷看他的计划了？"
"没有。是他贴在白板上的。"
刘韧笑了。然后他认真地说："如果他选助教，应该是你。"
"不一定。你基础比我好。"
"基础好不等于能教。"刘韧说。"你能让赵天参考你的代码、还能让他心服口服地在注释里标出处——这比基础好重要。"
那天晚上。903室。
马农在人事评估文件里加了一行。不是在林夕名字后面——是在文件的最上方：
`
Day 13: 林夕完成了第一段超出要求的独立代码。
她开始理解"工程"和"编码"的区别了。
可造之材。确认。
`
然后他翻到了培训计划的第四周。
`
Day 28: 选一名学员做助教，协助第二批培训
候选人：___
`
他在横线上写了两个字。
`
候选人：林夕
`
然后他想了想，在旁边加了括号：
`
候选人：林夕（备选：刘韧）
`
他合上了文件。
打开了903系统的监控：
`
Day 28 系统状态：
- 在线用户：5,834
- 日消息量：51,000+
- 新增服务：配置管理（老李完成v1.0）
- 待建服务：17
- 可维护人员：2（马农+老李）
`
两个人了。
老李上周学完了马农写的配置管理服务的代码，现在可以独立维护。不是老李参加了编程班——他本来就是后端工程师，只是CHEP前已经很久不写代码了。马农给了他代码和文档，他花了三天读完，然后说"我接手"。
两个人。比一个人好。
但还远远不够。
深夜。马农翻开师父的笔记。翻到了第十五页。
"我收学生的标准只有一条：他做的事情比我要求的多。
不是多做很多。是多做一点。一个方法、一个检查、一个他本来不需要写但写了的功能。
这'多出来的一点'不是勤奋。是好奇心。
好奇心不能教。你要么有，要么没有。有的人教他一，他做一。有的人教他一，他做一点五。
那个0.5，就是天赋的全部含义。"
马农看着这段话。
今天。林夕做了一点五。
她做了他要求的物资管理系统。然后多做了日志功能。然后多留了一个接口给未来。
0.5。
师父。你说的对。
马农合上了U盘的文件。
他走到窗前。
深圳的第二十八个夜晚。月亮出来了——CHEP以来第一次看到月亮。不是因为天气。是因为空气清洁了。没有工厂、没有汽车尾气、没有数据中心的冷却塔排放的水蒸气遮挡天空。
空气清洁了。因为文明暂停了。
马农看着月亮。
他想到了师父去世那天。也是有月亮的。
他想到了师父的最后一页笔记——那八个字。
代码的尽头是人。
今天他看到了。在林夕的代码里。在她多写的那个日志功能里。在她留的那个TODO注释里。
代码不是写给编译器的。是写给人的。
林夕理解了。
马农关了窗。
在培训计划上画了第十三天的勾。旁边写了三个字：
`
薪火传。
``
然后关了灯。
明天开始第三周。网络编程。Socket。
从明天开始，他们不只是在写练习了。他们要写能真正跑在903系统上的代码。
真刀真枪的开始。
