第26章 深圳-广州
一百公里的光纤里，流淌的不是数据。是希望。
CHEP后第三十天。
古法编程班第三周。网络编程。
消息是老李带来的。
"赵刚回来了。"老李推开903室的门，手里拿着一张手写的报告。"光纤通了。"
赵刚。三十五岁。基础设施工程师。CHEP后一直在做物理线路的事——光纤熔接、无线网桥、柴油发电机维护。他不参加编程班。他的战场在地下管道和楼顶天线上。
两周前，马农给了他一个任务：沿着广深高速的光纤管道做物理检查，找到一条从深圳到广州还能用的裸光纤链路。
这不是一个简单的任务。
CHEP后，广深高速的智能交通系统全面瘫痪。路面上散落着冻在原地的自动驾驶车辆。管道检查需要人工进入地下光纤井，用光功率计逐段测试信号衰减。
赵刚带了两个人，开着柴油越野车，花了十二天。
"从宝安到东莞到广州——我们测了四十七段光纤。"赵刚的报告写在一张A3工程图纸上，密密麻麻。"三十九段断了。或者衰减太大。但有八段是通的。"
他在图纸上画了一条红线——从深圳B栋数据中心到广州天河区的一个移动通讯机房。中间经过东莞的两个节点。
"这八段拼起来，总长107公里。信号衰减在可接受范围内。"
"延迟呢？"马农问。
"物理层没问题。延迟取决于你在两端放什么设备。"
马农点头。
"今天。"他说。"今天我们让这条线跑起来。"
马农把编程班的课改了。
原定的第三周课表是Socket编程基础。现在变成了实战——在真实的物理光纤上建立深圳到广州的第一条通讯链路。
"你们不再是在写练习了。"马农站在白板前，面对十一个学员。"今天你们写的代码会被部署到真实的服务器上，跑在真实的光纤上。从深圳发一条消息到广州——107公里。"
所有人都坐直了。
赵天悄声问刘韧："我们真的要上了？"
"他说的'不再是练习'，你觉得是开玩笑？"
马农在白板上画了架构图：
``
深圳 B栋 903室                     广州 天河机房
┌──────────────┐    光纤107km    ┌──────────────┐
│  Server (TX)  │═══════════════│  Server (RX)  │
│  端口: 9903   │               │  端口: 9903   │
│  Java Socket  │               │  Java Socket  │
└──────────────┘               └──────────────┘
`
"Socket通讯——飞鸽传书。"马农说。"TCP协议在两台机器之间建立一条可靠连接。一端发，一端收。光纤是路。Socket是车。TCP是交通规则。"
他在白板上写了核心代码：服务端用ServerSocket(9903)在端口监听，accept()阻塞等待连接；客户端用Socket("10.0.107.1", 9903)发起TCP三次握手——SYN、SYN-ACK、ACK，三次握手成功，连接建立，然后就可以往里写数据了。
他在白板旁边写了一个比喻：
`
TCP三次握手 = 打电话
SYN:     "喂？听得到吗？"
SYN-ACK: "听得到。你呢？"
ACK:     "我也听到了。开始说话。"
`
林夕在笔记本上写下了这个比喻。旁边画了一个小箭头。
"但不是写个Socket就能通的。"马农说。"从深圳到广州，中间有四个问题。"
他在白板上列了出来：
`
问题1：IP地址——光纤两端的服务器需要手动配置IP
问题2：路由——中间经过东莞两个节点，需要静态路由
问题3：可靠性——光纤信号不稳定，连接可能随时断
问题4：编码——双方必须约定消息格式，否则收到的是乱码
`
"今天上午，我负责前两个——配IP和路由。你们负责后两个——写一个可靠的通讯客户端。"
他在白板上给了需求：
`
任务：写一个可靠的Socket通讯客户端
要求：
1. 连接失败时自动重试（最多5次，每次间隔3秒）
2. 发送消息前加时间戳和发送方标识
3. 收到消息后打印并确认（ACK机制）
4. 连接断开时自动尝试重连
5. 所有通讯记录写入本地日志文件
`
"两个人一组。分工。一小时。"
林夕和刘韧一组。
他们的分工很自然——刘韧写网络连接和重试逻辑，林夕写消息格式和日志记录。
刘韧写了一个重连逻辑——for循环重试5次，每次间隔3秒，失败打印当前尝试次数。林夕的消息格式用|分隔时间戳、发送方、内容。
马农巡视的时候看了他们的代码。
"消息分隔符。"他指着|。"如果消息内容里也有|呢？"
林夕想了一秒："内容里的|需要转义……或者用一个不太可能出现在正文里的分隔符？"
"用\u0001。"马农说。"ASCII控制字符。正常文本里不会出现。"
他在白板上加了一行：
`
消息协议v1.0：
分隔符: \u0001 (SOH, Start of Heading)
格式: 时间戳\u0001发送方\u0001内容
`
"这就是协议设计的第一步——定义消息边界。"马农说。"TCP只保证字节按顺序到达。它不告诉你一条消息从哪开始、到哪结束。你必须自己定义。"
上午十一点。
马农和赵刚完成了物理层配置。
深圳端的服务器：10.0.107.100。
广州端的移动机房里有一台还能用的服务器（赵刚在路上摸清了的）：10.0.107.1。
静态路由通过东莞两个光纤节点中继。
马农亲自写了一个最简单的服务端程序——部署在广州端。一个ServerSocket，监听9903端口，收到消息打印并回复"ACK"。
他通过赵刚留在广州的一个对讲机频道，遥控广州端的一个志愿者启动了服务端程序。
"广州那边说——服务器跑起来了。等连接。"老李拿着对讲机说。
马农转向学员们。
"谁来连？"
十一双眼睛看着他。
"林夕。"马农说。"用你和刘韧的客户端。"
林夕的手在抖。
不是冷。九月末的深圳还很热。
她坐在终端前。屏幕上是她和刘韧写的ReliableClient。编译通过了。本地测试通过了——用localhost连自己。但现在要连的不是localhost。是107公里外的广州。
java ReliableClient 10.0.107.1 9903
她按下了回车。
屏幕上打印：尝试连接 10.0.107.1:9903 ...
三秒。五秒。十秒。
所有人都在看她的屏幕。903室安静到能听见光纤熔接机在角落里待机的蜂鸣声。
连接失败 (1/5), 3秒后重试...
林夕的心沉了一下。
"正常。"马农的声音从她身后传来。很平。"可能是路由还没收敛。等。"
三秒后。第二次尝试。又是十秒的等待——连接失败 (2/5)。
赵天在旁边小声说："不会是光纤断了吧？"
赵刚拿起对讲机："广州，光功率计什么读数？"
对讲机里传来断断续续的声音："……-8dBm……正常范围……"
信号没问题。
马农看了一下网络配置。他打开另一个终端，输入：ping 10.0.107.1
通了——64 bytes, seq=1, ttl=62, time=3.2ms。
"ping通了。Socket连不上。"马农说。"防火墙。"
他拿起对讲机。
"广州。检查iptables。9903端口是否放行。"
一分钟后。对讲机："……放行了。重试。"
林夕看着屏幕。第三次重试正在进行——三秒后，屏幕上跳出：连接成功: 10.0.107.1:9903
903室爆发出一阵欢呼。
"安静。"马农说。
欢呼声收了。但每个人的脸上都绷不住笑意。
"发消息。"马农对林夕说。
林夕在终端里输入：903室呼叫广州。深圳数据中心，CHEP后第三十天。一切正常。收到请回复。
她按下回车。屏幕上显示发送成功，然后等待回复。
五秒后，屏幕上跳出一行字：收到！这里是广州移动机房。有六个人。活着。缺水。听到你们的消息非常高兴。
林夕没有说话。
她的眼泪掉了下来。
马农走到她身边。他没有安慰她。他只是说了一句：
"继续。问他们需要什么。"
林夕擦了一下眼睛。输入：你们的情况？需要什么支援？
回复来得很快：六人。三男三女。粮食够一周。水源紧张。发电机还能跑。但柴油不多了。最大的问题：完全没有通讯。你们是第一个联系到我们的。
903室里所有人都在看屏幕。107公里外的六个人。CHEP后三十天。第一次有人联系他们。
林夕输入：我们会想办法送物资。保持这个频道开着。每天上午十一点通讯一次。
回复：好。谢谢。谢谢你们。
下午。
马农在白板上总结了今天的技术点：
`
今日复盘：
1. TCP三次握手——建立可靠连接
2. ServerSocket/Socket——Java网络编程基础
3. 消息协议设计——分隔符/时间戳/ACK
4. 重连机制——connect失败不是终点
5. 防火墙——物理层通了不等于应用层通了
`
"今天你们学了Socket。"他说。"但今天真正重要的不是Socket。"
他指着白板上的通讯记录——林夕和广州之间的六条消息。
"是这个。"
"107公里。一条光纤。两台服务器。一段你们写的代码。六个人知道了——他们不是一个人。"
他放下笔。
"这就是编程的意义。不是写for循环。不是背API。是连接人。"
那天晚上。
马农在终端前。他在给广州的通讯程序加功能——消息队列、离线缓存、断线自动重发。
林夕写的客户端是v1。能用。但不够稳。107公里的光纤链路会有信号抖动。需要心跳检测——每30秒发一个空包，确认连接还在。需要消息确认——发送方等待ACK，超时重传。
这些功能他可以自己写。但他决定——留一部分给林夕。
他在ReliableClient`的代码旁边加了三个TODO：心跳检测、消息确认、断线重连——每个都是林夕接下来要做的功课。
然后他保存了文件。
窗外。深圳的第三十个夜晚。
107公里外的广州，有六个人第一次在三十天里知道了——外面还有人。
马农关了灯。
