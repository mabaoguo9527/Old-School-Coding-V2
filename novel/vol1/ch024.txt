第24章 薪火

代码的尽头是人。

CHEP后第二十四天。

古法编程班第九天。面向对象·实践课。

十一个人。

少了一个——外包运维张伟。他昨天留下一张纸条："对不起，我要回老家。走陆路。家人在湖南。"没有人拦他。在这个世界里，选择回家不需要理由。

马农看了一眼纸条。"嗯。"没了。

今天的内容是"类与对象"的实践——让学员们真正用class写一段有结构的代码。

马农在白板上出了一道题：

``
练习：设计一个"服务器监控"系统
- Server类：属性包括名称、IP、CPU使用率、内存使用率、状态（运行/停止）
- 方法：启动、停止、打印状态
- ServerGroup类：管理多台Server
- 方法：添加服务器、打印所有服务器状态、找出CPU使用率最高的服务器
`

这道题不是随便出的。马农给的每一道练习题都有实战背景——这道题就是903室服务器监控系统的简化版。

"一个小时。"

学员们开始写。

四十分钟后，马农在巡视的时候看到了刘韧的代码：

`java
public class Server {
    private String name;
    private String ipAddress;
    private double cpuUsage;
    private double memoryUsage;
    private boolean running;
    
    public Server(String name, String ipAddress) {
        this.name = name;
        this.ipAddress = ipAddress;
        this.running = false;
        this.cpuUsage = 0.0;
        this.memoryUsage = 0.0;
    }
    
    public void start() {
        this.running = true;
        System.out.println(name + " 已启动");
    }
    
    public void stop() {
        this.running = false;
        System.out.println(name + " 已停止");
    }
}
`

规范。干净。字段private，方法public——封装做得对。命名自解释。

"可以。"马农说。

然后他看到了林夕的代码。

她的Server类多了一个东西——一个toString()方法：

`java
@Override
public String toString() {
    return String.format("[%s] %s | CPU: %.1f%% | MEM: %.1f%% | %s",
        name, ipAddress, cpuUsage, memoryUsage,
        running ? "运行中" : "已停止");
}
`

"你怎么知道toString的？"马农问。

"JDK文档。Object类的方法列表里看到的。"林夕说。"我猜如果覆写这个方法，System.out.println直接打印对象的时候会调用它。"

"你猜对了。"

马农没有多说。但他在白板上加了一行：

`
面向对象的三个直觉：
1. 数据和行为在一起（类）
2. 外面不需要知道里面怎么实现的（封装）
3. 同样的接口，不同的行为（多态）← 第三周
`

"林夕用的toString就是第三点的一个例子。Object定义了toString。你的Server覆写了它。JVM在打印的时候调用的是你写的版本——不是Object的默认版本。同一个方法名，不同的行为。这就是多态。"

他看了林夕一眼。"第三周的内容，你提前碰到了。"

一个小时结束。九个人完成了。赵天和产品经理陈莉还差一点——ServerGroup的"找最高CPU"方法没写完。

马农没有批评。他说了一句："回去写完。明天早上交。"

然后他擦掉了白板。

学员们以为要讲新内容。

马农在白板上写了一行字：

`
故事时间。
`

所有人愣住了。

九天了。马农从来没讲过任何与代码无关的事情。

他在白板旁边坐下。不是站着——是坐在那张折叠椅上。这是另一个反常的信号。他上课永远站着。

"今天不讲技术。"他说。"讲一个人。"

903室安静了。连键盘声都没有了。

"我的师父。"

马农的师父没有名字。

不是没有名字——是马农从不提他的名字。在马农的叙述里，他永远只是"师父"。

"2015年。我十五岁。高一。"马农说。语速比平时慢。句子比平时长一点点。

"那时候还没有Origin。代码是人写的。IDE是Eclipse。版本控制是SVN。你们大部分人没听过这些——正常。那个时代已经消失了。"

"我在深圳一个线下技术社区认识了他。那种社区现在没有了——一群人每周六在一个咖啡馆二楼聚会，带着笔记本电脑，互相看代码。没有线上、没有直播、没有弹幕。面对面。"

"他七十多岁。退休了。八十年代在航天系统写汇编。手写代码写了四十年。他不上网。用一台旧ThinkPad，终端永远停在vim。"

马农停了一下。

"第一次见面。我带了一段自己写的程序——一个简单的排序算法。我觉得挺好的。写了大概五十行。"

"他看了三秒。"

"然后说：'不行。'"

有几个学员笑了。

"我问他哪里不行。他说：'全不行。'"

笑声更大了。但马农没有笑。

"然后他说了一句话。这句话我记了二十三年。"

他停顿了两秒。

"'你不配写代码。先读十万行。'"

903室安静了。

"他让我读代码。不是写——是读。"马农说。"他给了我一份Linux内核的源码，打印出来的。三千多页。他说：'从第一页开始，逐行读。不懂的记下来。一周后来找我。'"

"我读了一周。记了二百多个不懂的地方。"

"他看了我的笔记。说：'嗯，你至少知道自己不懂什么。大部分人连这个都不知道。'"

"然后他又给了我一份——JDK源码。java.util.HashMap的实现。两千多行。他说：'把每一行的作用写出来。为什么用这个数据结构，为什么是这个初始容量，为什么负载因子是0.75。'"

"我花了一个月。写了一万字的分析笔记。"

"他看完之后说了两个字：'继续。'"

"就这样过了两年。"马农说。"两年。我没写过一行自己的代码。全在读。"

"读Linux内核、读JDK源码、读GCC编译器的代码、读TCP/IP协议栈的实现。他给什么我读什么。"

"两年后的一天。他说：'写个东西给我看。'"

"我写了一个简单的内存池——手动管理内存分配和释放。C语言。大概三百行。"

"他看了五分钟。逐行看。"

"然后他说：'还行。'"

马农的嘴角动了一下。几乎看不出来。

"'还行'是他给过我的最高评价。"

"2017年。他去世了。"

马农的声音没有变化。平静。像在描述一个技术问题的解决方案。

"心脏病。很突然。我接到电话的时候在学校。赶到医院的时候已经走了。"

他停了三秒。

"他的遗物只有两样东西留给我。一台旧ThinkPad——终端还停在vim，光标在第47行。第47行写了半个函数，还没写完。"

"另一样是一个U盘。里面有一份文件——他写了四十年的笔记，浓缩成三万字。标题叫《手写的温度》。"

马农看着白板。白板上什么都没有——他刚才擦掉了所有代码。

"那份笔记的最后一页只有一句话。"

他站起来。走到白板前。拿起马克笔。

写了八个字：

`
代码的尽头是人。
`

903室没有人说话。

不是那种尴尬的沉默。是另一种——每个人都在消化这八个字的重量。

林夕的眼睛红了。她没有哭出来——但她低下了头。

马农看着白板上的字。

"你们知道他为什么教我读代码而不是写代码吗？"

没人回答。

"因为写代码是技术。读代码是理解。技术可以被工具替代——Origin就替代了99%。但理解不能。"

"你不理解一段代码为什么这么写，你就不可能改进它。你不理解一个系统为什么崩溃，你就不可能修复它。你不理解一个人为什么需要这段程序，你就不可能写出真正有用的东西。"

他放下马克笔。

"Origin能写代码。但Origin不理解代码。它生成的每一行都是统计学上最可能正确的组合——不是因为它理解问题，而是因为它见过足够多类似的问题。"

"这就是它崩溃的根本原因——当它遇到从未见过的情况时，统计学失效了。概率模型崩塌了。而理解不会崩塌。因为理解是你自己的。"

他看着十一个人。

"我今天教你们的不是面向对象。不是ClassLoader。不是for循环。"

"我教你们的是——理解。"

"你们可以忘掉我教的每一行代码。但不要忘记这八个字。"

他指着白板。

`
代码的尽头是人。
`

那天下课很安静。

没有人问问题。没有人互相讨论。十一个人各自收拾东西，各自离开。

林夕是最后走的。

她站在白板前。看了那八个字很久。

然后她拿出笔记本——第27页——把这八个字抄了下来。

不是抄在代码笔记的区域。是抄在本子的内封面上。用红笔。

那天晚上。马农一个人在903室。

他打开了师父的U盘。《手写的温度》。三万字。

他不常看这个文件。不是因为不想看——是因为每次看都会想起某些东西。

他翻到了第七页。师父写的一段话——关于教学的：

"我这辈子教过七个学生。只有两个坚持到了最后。一个去了航天，一个去了腾讯。去航天的那个现在退休了，偶尔给我寄茶叶。去腾讯的那个从来不联系我。但我知道他还在写代码。因为他是我教过的人里，唯一一个问'为什么'比问'怎么办'多的人。"

马农看了这段话。

"去腾讯的那个"是他自己。

他确实从来没有联系过师父——不是不想。是不知道说什么。师父教了他两年，他用了十七年。师父给他的那些代码阅读任务，他后来在腾讯的每一次排障中都用到了。

师父不知道。

师父走的时候不知道。

马农翻到了最后一页。那句话。

"代码的尽头是人。

这句话有两层意思。第一层：代码是为人服务的，不要写只有机器能读懂的代码。第二层：代码最终要被传承的，不要写只有你自己能维护的代码。

你写的每一行代码，背后都站着两个人——现在使用它的人，和未来维护它的人。如果你只为现在的人写，代码会在你离开后死掉。如果你也为未来的人写，代码就能活得比你久。

这就是手写的温度。不是温度高。是温度在。

活的代码有人的温度。死的代码只有逻辑。"

马农合上了文件。

他看着窗外。

深圳的第二十四个夜晚。没有灯。但903室里有。

十五台服务器。十一个学员。一块白板。一支马克笔。

师父教了他。他在教林夕们。

薪火。

不是因为火要灭了才传。是因为一个人的火照不亮整条路。

他打开终端。不是打开监控——是打开了一个新的文件。

`bash
vi ~/docs/teaching-notes.md
`

标题：

`
古法编程·教学笔记
记录教学过程中的观察和反思
如果有一天我不在了，接手的人看这个
`

他写了第一条：

`
Day 9 观察：
林夕自己翻了JDK文档找到了toString。这不是教出来的。
刘韧的封装做得对但没有惊喜。稳定但不突破。
赵天在进步。命名的问题改了七成。态度比能力重要。

教学反思：
师父说"你不配写代码先读十万行"。
我没有两年时间给他们。
所以我必须把两年的东西压缩成四周。
代价是：他们的基础不会像我当年那么扎实。
好处是：他们会更快进入实战。
这是一个取舍。但在CHEP的世界里，没有完美的选择。只有"来得及"和"来不及"。
`

他保存了文件。

然后在培训计划上画了第九天的勾。旁边写了一行字——用了和之前不一样的笔：

`
今天讲了师父的事。
第一次对别人讲。
不知道为什么。也许因为——
今天教面向对象的时候，看到林夕自己去翻文档，
想到了师父说的那句话："唯一一个问'为什么'比问'怎么办'多的人。"
``

他停了笔。

看了这段字一秒。然后把本子合上了。

凌晨一点。

903室的灯关了。

白板上的八个字在黑暗中看不见了。但它们在那里。

就像师父的声音在马农的耳朵里。看不见，但一直在。
