第13章 手术刀与铁锤

能用手术刀的时候，不要用铁锤。

CHEP后第五十八小时。

数据中心的柴油在凌晨六点四十三分耗尽了。

老李打来电话的时候，马农正在写代码。他接起来，听老李说了一句"数据中心断电了"，看了一眼存储桥接的日志——最后一条rsync传输在六点四十一分完成——然后说了一个字："嗯。"

该搬的数据都搬了。BGP配置备份在笔记本硬盘上。前海数据中心的三千台服务器在断电的瞬间全部停机。那些一直空转的Origin推理引擎终于停了下来。

不是马农关的。是柴油替它们做了决定。

"数据中心那边还有人吗？"马农问。

"保安和设施运维还在。断电以后空调也停了，机房温度在升。他们说再待一两个小时就得撤。"

"让他们撤。数据中心的使命完成了。"

马农挂了电话。

他现在面对的问题很现实：903室的十五台服务器加上从数据中心搬回来的数据，构成了深圳——或许也是华南——唯一运行中的可用计算集群。但这个集群的能力极其有限。十五台服务器，总共240核CPU、512GB内存、48TB磁盘。

听起来很多。但这些资源要承担的工作包括：

1. 整个南山区（约180万人口）的基础通讯路由
2. DNS解析（没有DNS，局域网内的设备也无法通过域名互访）
3. 存储服务（从数据中心搬回的业务数据）
4. 基础监控
5. 新的认证服务（马农昨天开始写的第四个应急服务）

五个服务跑在十五台机器上——从绝对数量看足够了。但问题不在于静态负载，在于动态增长。

随着更多人知道"南山B栋有可用网络"，接入请求会越来越多。马农昨天教的那七个工程师里，有两个已经开始帮忙搭建二级路由节点——但他们的技术能力只够照搬马农写好的配置，无法独立应对异常情况。

马农需要更多的服务。更稳定的服务。

他需要把五个手写的应急Java程序，从"能跑"升级到"能扛"。

上午九点。白板前。

马农擦掉了之前的内容，写了一个新的标题：

``
应急服务架构升级计划
`

七个工程师围在白板旁边。有些人端着方便面纸碗——物资来自赵刚去超市"以物换物"搞回来的一箱泡面。

马农在白板上画了一张图。不是PPT——他这辈子都不用PPT。是架构图。用黑色马克笔。线条简洁，没有花哨的图标，只有方框、箭头和关键标注。

`
当前架构（v1 应急版）:
┌──────────┐
│ Router   │─→ 单线程accept, 线程池处理
├──────────┤
│ DNS      │─→ 内存缓存, 无持久化
├──────────┤
│ Monitor  │─→ 轮询式, 5s间隔
├──────────┤
│ Storage  │─→ rsync桥接, 无本地冗余
├──────────┤
│ Auth     │─→ 未完成
└──────────┘

问题：
1. Router单点故障 → 全网瘫痪
2. DNS无持久化 → 重启丢失
3. 无服务间健康检查
4. 无优雅降级策略
`

他放下笔。看着七个人。

"这五个服务是我在过去四十八小时里用vi一行一行敲出来的。"他说。"它们能跑。但它们不能死。"

"任何一个服务死了——路由挂了，整个网络断开；DNS挂了，没人找得到谁；存储挂了，数据没了。这不是'修好就行'的问题。是'不能出错'的问题。"

他指着白板上的"问题"列表。

"今天的任务：把v1升级到v2。不是重写。是加固。"

马农分配了任务。

七个人，按能力分三组。

A组（老李+赵刚）：给Router加一个备用实例。两台服务器各跑一个Router，用VRRP（虚拟路由冗余协议）做主备切换。主的死了，备的自动接管。

马农在白板上给他们画了VRRP的原理图——一个虚拟IP，两台实体机竞选Master。Master每秒发一次心跳广播，Backup监听。心跳断了三次，Backup升级为Master。

"VRRP。"马农说。"你们不需要理解RFC 5798的全部内容。你们只需要理解一件事：两个人轮流站岗。一个倒了，另一个自动上。"

赵刚点头。"像哨兵换岗。"

"对。"

B组（小陈+另外两个年轻工程师）：给DNS加持久化。当前的DNS服务把所有记录存在内存里——JVM重启就全没了。马农要求他们把DNS记录同步到本地文件。每次更新内存的同时写一份到磁盘。启动时先从磁盘加载。

马农给他们写了一段示例代码：每次put写入内存的同时调用persistToDisk()同步落盘，用Files.write把整个cache写成文本文件。关键在于catch住IOException但不抛异常——落盘失败不影响内存服务，降级但不停机。

"注意最后那行注释。"马农说。"DNS的首要任务是解析域名，不是保存文件。磁盘坏了也要继续解析——这叫优雅降级。能做的做，做不了的放弃，但服务不能停。"

小陈在笔记本上记。

C组（剩下两人）：写一个健康检查服务。每三十秒ping一次所有服务端口。如果连续三次ping失败，在监控面板上标红，并且往马农的终端上发一条告警。

马农没给C组写示例代码。他说："自己写。用Socket连目标端口，连上了就是活的，连不上就是死的。三十行以内能搞定。"

两个人面面相觑。其中一个——二十六岁的前端工程师，名叫王浩——小声说："马……马农，我从来没用Java写过Socket。"

"那你学过什么？"

"HTML、CSS、Origin提示词……"

"Socket的本质是一个文件描述符。"马农说。"你往里面写字节，对面收到字节。TCP保证顺序和完整性。你只需要知道：new Socket(host, port)——连上了就是活的。"

他在白板上写了两行：

`java
try { new Socket(host, port).close(); return true; }
catch (Exception e) { return false; }
`

"就这些。其他的自己想。"

马农自己做的是最关键的一件事：写认证服务。

Auth——认证。在CHEP之前，每一个用户的身份验证都由Origin管理。登录、权限、会话、token——全套。Origin停了以后，没有认证系统意味着任何人可以访问任何服务。在903室的小局域网里这不是问题。但如果网络扩大，接入的人越来越多，没有认证就是裸奔。

马农打开vi。

`bash
vi ~/emergency/auth/SimpleAuthService.java
`

他写了一个极简的认证服务。不用Spring Security，不用OAuth2，不用JWT——这些框架层的东西在没有Origin、没有Maven仓库、没有网络下载依赖的环境下，用不了。

他用的是最原始的方式：用户名+密码的SHA-256哈希比对+基于时间戳的会话token。

`java
public class SimpleAuthService {
    
    // 用户凭证存储（明文密码不存储，只存哈希）
    private final Map<String, String> credentials = new ConcurrentHashMap<>();
    
    // 会话管理（token -> 过期时间）
    private final Map<String, Long> sessions = new ConcurrentHashMap<>();
    
    private static final long SESSION_TTL = 3600_000; // 1小时
    
    public String authenticate(String username, String password) {
        String storedHash = credentials.get(username);
        if (storedHash == null) return null;
        
        String inputHash = sha256(password);
        if (!storedHash.equals(inputHash)) return null;
        
        // 生成session token
        String token = sha256(username + System.nanoTime());
        sessions.put(token, System.currentTimeMillis() + SESSION_TTL);
        return token;
    }
    
    public boolean validate(String token) {
        Long expiry = sessions.get(token);
        if (expiry == null) return false;
        if (System.currentTimeMillis() > expiry) {
            sessions.remove(token);  // 过期清除
            return false;
        }
        return true;
    }
    
    private String sha256(String input) {
        try {
            MessageDigest md = MessageDigest.getInstance("SHA-256");
            byte[] hash = md.digest(input.getBytes(StandardCharsets.UTF_8));
            return bytesToHex(hash);
        } catch (NoSuchAlgorithmException e) {
            throw new RuntimeException(e);  // SHA-256必存在
        }
    }
}
`

一百二十行。从类声明到HTTP端点到会话管理到密码哈希。

他写的时候不查文档。不翻书。不犹豫。

手指在键盘上的节奏均匀而快速，像一个外科医生在做最熟悉的手术——每一刀的位置、深度、角度都是肌肉记忆。

老李路过的时候看了一眼他的屏幕。代码以肉眼可见的速度在增长。没有回退。没有删除。每一行写下去就是对的。

"你写代码的时候不思考吗？"老李忍不住问。

马农没抬头。"思考在写之前完成。动手只是输出。"

他说这句话的时候，手指没有停。

下午两点。A组和B组分别完成了任务。

Router的VRRP备份经过马农检查，确认了主备切换的逻辑和心跳超时参数。DNS的持久化经过马农测试——他手动kill掉DNS进程再重启，确认记录从磁盘恢复成功。

C组的健康检查服务还在写。王浩写了三版都没过。第一版连接超时没设——Socket连一个死端口会等默认的几十秒才返回。第二版设了超时但没关Socket——连接泄漏。第三版关了Socket但逻辑写反了——活的报死，死的报活。

马农走过去看了一眼。

"读代码。"

"啊？"

"读你自己写的代码。出声读。每一行。"

王浩尴尬地读了起来。读到第十七行的时候——if (!connected) return true——他自己停住了。

"这里……好像写反了。"

"嗯。"

"应该是if (connected) return true。"

"嗯。"

马农走了。

王浩的第四版通过了。二十三行。比马农说的三十行还少。

傍晚六点。全部服务升级完毕。

马农在白板上更新了架构图：

``
v2 加固版:
┌──────────┐  ┌──────────┐
│ Router-A │←→│ Router-B │  VRRP主备
├──────────┤  └──────────┘
│ DNS      │─→ 内存+磁盘持久化
├──────────┤
│ Monitor  │─→ 轮询+端口健康检查
├──────────┤
│ Storage  │─→ 本地副本+日志审计
├──────────┤
│ Auth     │─→ SHA-256+Session Token
└──────────┘
`

他站在白板前看了一会儿。

不完美。很多设计在正常的开发环境里会被嘲笑——没有负载均衡、没有服务发现、没有配置中心、没有分布式事务。这些东西在有Origin的时代一行提示词就搞定了。

但现在没有Origin。

有的只有vi、javac、十五台服务器、和八个能干活的人。

马农在架构图旁边写了一行小字：

`
手术刀，不是铁锤。
解决当下问题。不过度设计。
``

他放下笔。

"今天到这。"他对七个人说。"明天还有事。"

七个人开始收拾。有人往嘴里塞最后一口泡面。有人在充电宝上给手机续命。小陈把今天学到的代码手抄在笔记本上——纸质笔记本，不是电脑。因为电脑随时可能没电，但纸不会。

马农独自坐回到SRV-1前。

他打开了认证服务的日志。系统运行平稳。第一个注册用户是他自己。用户名：root。密码的哈希值他记在脑子里。

这座城市断网的第三天。

马农的手写代码在十五台服务器上安静地运行着。路由分发着流量。DNS解析着域名。监控检查着心跳。认证校验着身份。存储保存着数据。

五个服务。八个人。一间四十平米的工作室。

这是深圳目前唯一的互联网。
