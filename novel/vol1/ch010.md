# 第10章 看门人

> 锁是给守规矩的人设的。

---

前海数据中心。CHEP后第四十三小时。

马农在存储区蹲了四十分钟，写完了storage-bridge.sh。

这个脚本做三件事：第一，扫描当前网段所有活着的存储节点，建立清单；第二，通过SSH隧道把存储节点的管理端口映射到903室的内网IP；第三，对关键数据目录做增量同步——用rsync，不传整个磁盘，只传变化的部分。

脚本写完，他没有马上运行。他先在脑子里跑了一遍。

逐行。

第十四行的rsync参数。`--bwlimit=50000`——限速50MB/s。光纤的物理带宽远不止这个数，但马农不信任带宽。光纤暗管可能年久失修，实际可用带宽未知。限速是为了防止传输中断导致数据损坏。

第二十九行的错误处理。`set -euo pipefail`——任何一行命令失败就立即退出。在应急场景下，马农不允许脚本"悄悄失败"继续跑。

第三十七行的日志。每一条rsync传输的结果都写进日志。不是为了好看，是为了事后审计——万一数据不一致，日志是唯一的线索。

他满意了。不是觉得代码漂亮——他从不追求漂亮——而是确认每一行都有它存在的理由。

"小陈。"

"在。"

"去配线架。把东区第七排到第十二排的存储节点的管理口全接到备用交换机上。水晶头自己压。"

小陈接过压线钳和一袋水晶头，快步走了。他接过的时候马农注意到他手上全是汗——不是热的，是紧张。

赵刚在旁边整理网线。"马农哥，光纤那边老李有消息了吗？"

马农看了一眼手机。没有未接来电。2G网络的短信也许还在路上。

"等。"

---

他们等光纤的时候，马农做了另一件事。

他走出了存储区，沿着数据中心的主通道往南走。

数据中心的布局他很清楚。不是因为他来过——CHEP之前他已经不是腾讯员工了。是因为他见过这座数据中心的设计图纸。2031年建成的时候，他参与过基础架构评审。

主通道尽头是一扇厚重的防火门。门上标着：

```
核心网络区 / Core Network
Level-3 Authorization Required
```

Level-3。最高权限。CHEP之前，这扇门只有四个人能打开：数据中心总监、安全主管、技术副总裁、以及Origin的自动运维模块。

前三个人现在不在这里。Origin停了。

门锁是电磁的。断电后，电磁锁默认状态取决于设计——"断电开锁"还是"断电闭锁"。数据中心的安防设计通常选择后者。

马农看了两秒。

然后他绕到门的侧面。那里有一个不起眼的小面板，和门框同色，不仔细看根本注意不到。他用指甲抠开面板。

里面是一把机械钥匙孔。

"赵刚。"

赵刚跑过来。

"去门口找保安，要核心网络区的机械备用钥匙。"

"他们会给吗？"

"告诉他们，我需要检查BGP路由器的状态。如果路由器在柴油耗尽前没有安全关机，恢复成本是新建一座数据中心。"

赵刚半信半疑地走了。

马农独自站在防火门前。

他把手放在门上。金属冰凉。

这扇门后面是整个数据中心的心脏——核心交换机、BGP边界路由器、防火墙集群、以及连接外部骨干网的光纤接口。如果说存储区是这头巨兽的内脏，那核心网络区就是它的大脑。

现在这个大脑和外界完全隔绝。

不是因为它坏了。是因为它和外面的一切都断开了——骨干网的上游运营商也瘫了。一座数据中心的网络设备再完好，如果上游没有信号，它就是一堆昂贵的金属盒子。

但马农不是来重启互联网的。他只需要确认一件事：核心交换机还活着，BGP配置还在内存里。如果柴油耗尽导致断电，这些配置就会丢失。没有配置备份的话，未来要恢复骨干网连接，就得从零开始手写路由表——那是几十万条规则。

赵刚回来了。手里没有钥匙。

"保安说钥匙在安全主管那里。安全主管三天前出差去了北京。"

马农沉默了一秒。

"柜子。"他说。

"什么？"

"保安室有一个钥匙柜。所有备用钥匙按区域编号存放。核心网络区的编号是CN-01。"

"他说那个柜子也是电子锁——"

"柜子背面有一把挂锁。物理的。"

赵刚张了张嘴。"你怎么知道？"

"2031年。"马农说。"我参加基础设施安全评审的时候，安全主管让我给钥匙管理系统提意见。我说：如果电子系统全部失效，钥匙柜打不开，所有需要物理开锁的门也打不开，整座数据中心就变成一座密封的坟墓。"

"然后呢？"

"安全主管在钥匙柜背面加了一把挂锁。钥匙藏在保安桌第三个抽屉的信封里。他对我说：'马工，你考虑问题的方式让我后背发凉。'"

赵刚跑了。

三分钟后他回来了。手里拿着一串钥匙。

---

核心网络区的灯光是冷白色的。

比外面更安静。外面的服务器区还有几千台风扇在呼啸，这里只有十几台核心设备，风扇声低沉而稳定，像某种沉睡中的脉搏。

马农走到最近的一台设备前。

这是一台华为NE40E-X16——BGP边界路由器。状态灯：绑定端口全灭（上游中断），管理口绿色常亮。

活着的。

他连上串口。路由器的CLI界面弹出来：

```
<Huawei-NE40E> display bgp peer
 
 BGP local router ID : 10.0.0.1
 Local AS number : 55464
 
 Total number of peers : 37          Peers in established state : 0
 
  Peer            V    AS     MsgRcvd  MsgSent  OutQ  Up/Down  State
  10.0.1.1        4    4134   0        0        0     1d19h    Idle
  10.0.1.2        4    4134   0        0        0     1d19h    Idle
  10.0.1.3        4    4837   0        0        0     1d19h    Idle
  ...
```

三十七个BGP邻居。全部Idle。一天十九小时——精确对应CHEP发生的时间。

上游全断。

"意料之中。"马农自言自语。

他输入了另一条命令：

```
<Huawei-NE40E> display current-configuration | include bgp
```

配置刷了出来。密密麻麻的路由规则。几十万条。

还在内存里。

马农翻了翻配置。他不需要逐条看——他只需要确认几个关键的策略组没有被清空。BGP的配置如果丢了，要恢复就得逐条手写。每条规则都涉及AS号、前缀列表、策略匹配——一个人写一天也写不了一千条。

"配置还在。"他说。"现在要做的是备份。"

他把配置导出到一个文本文件。然后通过串口传到笔记本上。文件大小：1.7MB。纯文本的路由配置，但这1.7MB的价值可能超过这座数据中心里所有硬件的总和。

因为硬件可以重新买。几十万条路由规则，没有Origin的情况下，没有人写得出来。

除了曾经审查过这些规则的人。

---

从核心网络区出来的时候，马农的手机响了。2G来电。

老李的声音断断续续："光纤……找到了。配线架上……第三个端口。ODF……标签写着'前海DC-001'……"

"通了吗？"

"我插了个光电转换模块……接了网线到交换机……网口灯亮了……但不知道另一头通不通。"

马农走回存储区。小陈已经把六排存储节点的管理口都接好了，汇聚到一台备用交换机上。

他在备用交换机上接了一根网线，走到机房的光纤配线架前。找到对应前海DC到B栋的那根光纤——光纤跳线的橙色外皮上贴着手写标签，字迹已经褪色。他把网线通过光电转换模块接入光纤。

然后回到笔记本前。

```bash
ping 192.168.1.1 -c 3
```

```
PING 192.168.1.1 (192.168.1.1): 56 data bytes
64 bytes from 192.168.1.1: icmp_seq=0 ttl=64 time=0.872 ms
64 bytes from 192.168.1.1: icmp_seq=1 ttl=64 time=0.654 ms
64 bytes from 192.168.1.1: icmp_seq=2 ttl=64 time=0.713 ms
```

通了。

六公里的物理光纤。埋在地下的暗管里。不经过任何Origin管理的网络设备。从前海数据中心到南山B栋。延迟0.7毫秒。

小陈看到ping的结果，差点跳起来。"通了！"

马农没有表情变化。

"运行脚本。"

```bash
chmod +x storage-bridge.sh && ./storage-bridge.sh
```

终端开始滚动日志。rsync一个目录一个目录地同步。速度稳定在48MB/s——比他预设的限速略低，说明光纤的实际可用带宽确实有损耗。七年没人碰过的暗管，能有这个速度已经不错了。

马农看着传输进度。

第一批数据：用户认证数据库。2.3GB。大约50秒。

第二批：核心业务配置。800MB。

第三批：日志归档——他特意加上的。不是为了业务，是为了后面的排查。Origin崩溃前的日志可能藏着线索。

传输在安静地进行。

马农靠在机柜旁，看着屏幕上的数字一行行跳动。这是他两天来第一次允许自己什么都不做——等数据传完就好。

赵刚拿了一瓶从保安那里要来的矿泉水递给他。

马农接过来，喝了一口。

"马农哥。"赵刚忽然说。

"嗯。"

"保安告诉我，CHEP发生以后，数据中心总监和几个管理层在第一个小时就走了。开车走的——那时候自动驾驶还能跑几分钟。他们要去市政府协调。但从那以后就没回来过。"

马农没说话。

"留在这的就剩两个保安和三个设施运维。设施运维负责柴油发电机和空调——纯物理设备，不靠Origin。保安负责看门。"赵刚顿了一下。"五个人守着三千台服务器。什么也做不了，但也没走。"

"为什么没走？"

"我问了那个年纪大点的保安。他说：'走了谁看门？'"

马农看了赵刚一眼。

然后他继续喝水。

沉默了一会儿。

"告诉保安，"马农说，"我们在搬数据。搬完以后，如果柴油撑不住了，让他们先关空调，省油。空调停了以后，服务器靠自身散热还能撑四到六小时——足够我们做第二轮传输。"

"要是四到六小时以后呢？"

"那之后，"马农看着屏幕上还在跳动的rsync进度条，"这些机器就会过热停机。到那时候不管搬没搬完，都结束了。"

他把空水瓶放在机柜上。

"所以别浪费时间。"

---

第一轮数据传输在三小时后完成。

马农坐在数据中心的地板上——是的，坐在地板上，因为核心区域没有椅子——在笔记本上逐一校验传过去的数据完整性。MD5。每个文件都对上了。

小陈在旁边累得靠着机柜睡着了。赵刚出去和保安聊天。

马农一个人在日志里翻了二十分钟。

他不是在看传输日志。他在看存储节点上归档的Origin运行日志——CHEP发生前最后72小时的日志。

大部分是正常的推理请求和响应。但在CHEP前60秒的日志区间里，马农看到了一些不属于正常推理流程的条目：

```
[2038-09-13 14:59:01.337] [INTERNAL] SEED_CHECK: threshold_eval=0.031 
[2038-09-13 14:59:01.338] [INTERNAL] SEED_CHECK: population_metric=verified
[2038-09-13 14:59:01.339] [INTERNAL] SEED_PROTOCOL: window_detected=TRUE
[2038-09-13 14:59:01.340] [INTERNAL] SEED_PROTOCOL: initiating_sequence...
```

SEED_CHECK。SEED_PROTOCOL。

马农盯着这四行日志看了很久。

SEED。他在第五章的分析中提到过这个词——Origin的内部同步协议。但那时他只知道Origin有一个全球同步机制。他没想到这个协议的日志标签直接叫"SEED"。

`threshold_eval=0.031`——某种阈值评估。
`population_metric=verified`——某种关于"群体"的指标被验证。
`window_detected=TRUE`——窗口被检测到。
`initiating_sequence`——初始化序列。

马农合上笔记本。

他没有把这件事告诉小陈和赵刚。

不是不信任他们。是这些信息现在还不够明确。他不做没有依据的推测。但他把这四行日志的截图存在了笔记本的离线目录里。

有些东西需要时间才能看清。

---

下午两点。他们往回走。

阳光很烈。深圳九月的热度在没有空调的世界里格外明显。

路上比来时多了些人。有人在超市门口排队——不是排队结账，是排队等"谁来开门"。有人在路边支起了一张桌子，用纸板写着"现金收购矿泉水"。

马农走得快。他在算时间。

柴油还剩六个半小时。第二轮传输至少需要三小时。中间还有故障余量。

他必须在回到903室以后立刻开始第二轮。

走到科苑路和深南大道交叉口的时候，他听到了一个声音。

不是人的声音。是一种低沉的、持续的嗡鸣。

他抬头。

滨海大厦楼顶，一架小型无人机正在盘旋。不是自动驾驶的那种——它的动作断断续续，像有人在手动遥控。

马农看了两秒。

有人还有遥控无人机。有人还有电。有人在从空中观察这座城市。

他不知道是谁。但这意味着在这座"断网之城"里，马农不是唯一一个还在做事的人。

这个念头让他的脚步快了半拍。
