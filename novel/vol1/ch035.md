# 第35章 前夜

> 检查装备的时候就是最紧张的时候。因为检查完了就没退路了。

---

CHEP后第四十四天。下午三点。

广州。天河区。

他们到了。

市第三人民医院。一栋二十层的灰白色建筑。大门口的自动玻璃门停在了半开的状态——有人用砖头卡住了。

小周跳下车。"我去叫人。"

五分钟后。一个穿着白大褂的中年女人快步走出来。白大褂已经不太白了——四十四天没有洗衣机。但她的眼神清醒、锐利。

"你是马农？"

"嗯。"

"我是ICU护士长李敏。谢谢你来。"

她的声音没有发抖。但她的手在抖——马农注意到了。她右手的指关节上有老茧。不是打字的老茧。是连续几十天手动按压气囊留下的。

---

李护士长带他们上了七楼ICU。

走廊里有一种特殊的气味——消毒液、汗水、和某种说不清的焦灼感。

"柴油快没了。"李敏边走边说。"发电机还能跑大概四十个小时。电力主要供给ICU的监护设备和照明。呼吸机已经全部停了——固件崩溃后我们就没法启动。"

"四十四天没有呼吸机？"赵天问。

"没有电动的。"李敏推开ICU的门。"但我们有手动的。"

ICU。十一张床。十一个病人。

每张床旁边都有一个人——护士或志愿者。他们手里握着一个蓝色的橡胶气囊，连着面罩扣在病人脸上。

一按。松。一按。松。

每四秒一次。二十四小时不停。

"我们排了四班倒。"李敏说。"每人两小时。但人不够。有些护士连轴转。"

马农看着那些手。

有些手上缠着纱布——因为连续按压导致了水泡和擦伤。有些人的手臂在微微发颤——肌肉疲劳。但没有人停。

一按。松。一按。松。

每四秒一次。

四十四天。

马农走到最近的一台呼吸机前。

它是一台德国产的Dräger Savina 300。工业级呼吸机。控制面板是黑的。电源灯不亮。

他打开了后面板。

主控板——基于STM32F429。ARM Cortex-M4内核。Flash 2MB。这和他昨晚在加油站看的参考手册一致。

"赵天。"马农说。"把工具箱拿来。"

---

接下来的三个小时。马农做了四件事。

第一：硬件评估。

他逐台检查了十一台呼吸机。型号一致，都是Dräger Savina 300。主控板完好——没有物理损坏。电源板完好。传感器板完好。

问题在固件。

"Origin在2036年推送了一个医疗设备固件升级。"马农检查主控板上的Flash芯片标签。"把呼吸机的控制逻辑从本地固件改成了'Origin辅助控制'——本地固件只负责底层硬件操作，高层控制逻辑由Origin的远程推理引擎实时计算。"

"CHEP之后，远程推理引擎死了。本地固件尝试连接Origin——超时。进入异常状态。异常处理逻辑——由Origin生成——本身也依赖Origin。死循环。"

他在随身带的笔记本上记录：

```
Dräger Savina 300 固件分析：
- 控制板：STM32F429 (Cortex-M4, 180MHz, 2MB Flash)
- 固件版本：v8.2.1-origin (Origin辅助版本)
- 故障原因：Origin远程推理引擎离线 → 本地固件进入异常处理 
  → 异常处理本身依赖Origin → 死锁
- 解决方案：擦除Origin辅助固件，写入纯本地版本
```

第二：查找备用固件。

"这台呼吸机在2035年之前跑的是什么固件？"马农问李敏。

"……我不知道。我是护士，不是工程师。"

"设备有维保记录吗？"

李敏带他去了设备科。一个灰尘满布的小房间。文件柜里有十一台呼吸机的维保档案——纸质的。

马农翻到了固件升级记录：

```
2035-06-15: 固件升级 v7.1.0 → v8.0.0 (Origin辅助版本)
2036-03-22: 固件升级 v8.0.0 → v8.2.1-origin (完全托管版本)
```

"v7.1.0。"马农说。"这是Origin之前的纯本地固件。如果能找到这个版本的二进制文件——直接刷回去。"

他翻遍了维保档案。没有二进制文件。纸质记录里只有版本号。

"有没有备份光盘或者U盘？"

李敏摇头。"设备科的电脑全部联网。固件都是在线升级的。Origin推送的。"

"本地没有备份。"

"没有。"

马农沉默了五秒。

"那就写一个。"

---

第三：评估手写固件的可行性。

马农在笔记本电脑上打开了STM32F429的数据手册。七百多页的PDF——离线版。

呼吸机的核心控制逻辑其实不复杂：

```
呼吸机控制循环（简化）：
1. 读取压力传感器数据
2. 根据设定参数（潮气量、频率、PEEP）计算目标压力曲线
3. 控制步进电机驱动活塞
4. 监测实际压力，闭环调节
5. 异常时触发报警（压力过高/过低、断电、传感器失败）
6. 回到步骤1
```

"核心是一个PID控制器。"马农说。"比例-积分-微分。控制理论里最基础的闭环控制算法。"

他画了一个框图：

```
设定值 → [PID控制器] → 步进电机 → 实际压力
                ↑                        │
                └────── 压力传感器 ←──────┘
```

"这个我能写。"马农说。"用C。裸机代码。不需要操作系统。"

"多长时间？"李敏问。

"写代码——一晚上。测试——需要一台不连病人的呼吸机做实验。你有空机器吗？"

"设备科有一台备用的。送修过一次。"

"带我去看。"

---

第四：制定明天的计划。

晚上八点。医院的一间会议室。

马农在白板上写了时间表：

```
Day 1（今晚-明早）：
- 马农：手写呼吸机固件 v1.0（C语言，裸机）
- 赵天：协助测试——负责监控电脑上的串口数据
- 小陈：准备11台呼吸机的硬件接口（拆后面板、找JTAG/串口）

Day 2（明天）：
- 上午：在备用呼吸机上测试固件
- 下午：如果测试通过——开始逐台刷机
- 每台刷机时间窗口：<30秒

关键约束：
- 刷机期间呼吸机必须断电
- 手动气囊必须在旁待命
- 每台刷完立即上电测试
- 一台失败就停下来排查，不继续
```

李敏看着时间表。

"一晚上写完固件……可以吗？"

"可以。"马农说。"呼吸机的控制逻辑不复杂。复杂的是安全边界——过压保护、传感器异常、电源中断恢复。这些都要写。"

他看着李敏。

"你的人今晚能撑住吗？"

李敏点头。"能。再撑一晚上。"

她的手还在抖。但她的声音没有。

---

晚上十点。设备科。

马农坐在备用呼吸机前面。笔记本电脑摊开。串口线连着呼吸机的调试口。`arm-none-eabi-gcc`编译器在终端里待命。

他开始写。

```c
/* 
 * Ventilator Firmware v1.0 (Manual)
 * Target: STM32F429 (Cortex-M4)
 * Author: Ma Nong
 * Note: 替代Origin辅助固件的纯本地版本
 *       只实现基础CMV模式（持续强制通气）
 */
```

他的手指在键盘上稳定地敲击。速度不快——嵌入式代码不能快。每一行都关乎硬件操作。一个寄存器地址写错了，步进电机可能失控。一个中断优先级配错了，传感器数据可能丢失。

赵天坐在旁边。他的任务是监控串口输出——马农每写完一个模块就编译、刷入备用机、看串口打印的调试信息。

"赵天。"马农敲了半小时后说。

"在。"

"明天刷机的时候，你负责计时。每台机器从断电到上电，不超过二十五秒。如果到了二十秒我还没完成——你喊我。"

"明白。"

"还有一件事。"马农没有抬头。他还在写代码。"如果明天出了意外——比如固件有bug，呼吸机上电后表现异常——你第一件事是什么？"

赵天想了三秒。"断电。恢复手动气囊。"

"对。先保命。再排查bug。永远先保命。"

赵天点头。

马农继续写。

窗外的广州和深圳一样黑。但这间设备科的灯亮着。一台笔记本电脑的屏幕亮着。串口线在桌上蜿蜒。

一行一行。

C语言。裸机。没有操作系统。没有框架。没有Origin。

从寄存器初始化到中断配置到PID控制器到安全保护。

从第一行开始。
