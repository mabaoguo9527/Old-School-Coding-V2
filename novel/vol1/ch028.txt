第28章 生存竞争

当资源有限时，技术就是权力。这才是马农最不愿意看到的。

CHEP后第三十四天。

903室的消息系统收到了一条不寻常的信息。

``
[接收] 2038-10-09 14:22 | 未知来源
发送方IP: 10.200.3.17
内容: 你们的通讯频段已被监听。建议加密。——友善提醒
`

马农看了三秒。

然后他做了一件事——关闭了903室对外的所有非必要端口。

"这条消息从哪来的？"老李问。

"不是从我们的网络内部。"马农检查了路由日志。"10.200段不是我们的地址。有人通过未知路径接入了我们的网络边界。"

"怎么可能？我们的网络是封闭的——"

"不完全封闭。"马农说。"903的消息系统对外暴露了HTTP端口。任何能触达我们IP的人都能发消息进来。"

他看了一眼那条消息。

"'友善提醒'。"他重复了一遍。"这说明三件事。第一，有人知道我们的存在。第二，他有技术能力接入我们的网络边界。第三，他选择了提醒而不是攻击。"

"好人？"老李试探地问。

"不确定。但他不蠢。"

马农在白板上画了一张新的地图。

不是技术架构图。是一张势力分布图。

`
CHEP后第34天·华南势力分布（已知）

[深圳] 903室 ← 我们
  - 5800+用户
  - 3个核心服务在线
  - 光纤通广州
  - 11名学员在训

[广州] 天河移动机房
  - 6人
  - 有发电机
  - 通过903光纤联通
  
[东莞] ?
  - 赵刚路上报告：有人在组织本地通讯网
  - 不明势力

[香港] ?
  - 未知状态
  
[珠海/中山/佛山] ?
  - 未知状态
`

"我们不是唯一恢复通讯的人。"马农说。

这个判断很快被证实了。

下午。赵刚从东莞发回了一份更详细的报告：

`
东莞松山湖。
有一个团队在运营一个独立的通讯网络。
规模：约800用户。
核心人物：不确定。据说是一个前华为的网络工程师。
他们有自己的服务器、发电机、和一套基于Linux的通讯系统。
重要：他们对我们的深广光纤链路很感兴趣。
有人向我打听903室的情况。
`

马农读完报告。没有表情变化。

"技术领主。"他说。

"什么意思？"林夕问。她下午来上课，正好听到了这个词。

马农在白板上写了三行：

`
CHEP后的世界秩序：
1. 掌握通讯能力的人 = 掌握信息的人
2. 掌握信息的人 = 掌握资源协调能力的人
3. 掌握资源协调能力的人 = 事实上的权力中心
`

"我们在做的事——恢复通讯、建立网络——不只是技术工作。"他说。"在一个没有政府、没有警察、没有货币体系的世界里，网络就是权力基础设施。谁控制网络，谁就控制信息。谁控制信息，谁就是这片区域的事实统治者。"

他看着林夕。

"你觉得这是好事还是坏事？"

林夕想了想："如果控制者是好人……就是好事？"

"没有'好人'这回事。"马农说。"只有制度和规则。好人不受制约，也会变成坏人。"

他在白板上加了一行：

`
903室的原则：
通讯网络不属于任何人。它属于接入它的所有人。
`

"这是我建这个网络时定的第一条规则。"他说。"我写了代码，我部署了服务器，我配了路由。但这个网络不是我的。"

"可是……如果别人不这么想呢？"刘韧问。"东莞那个团队如果想抢我们的光纤线路呢？"

"所以我们需要防御。"

马农用了整个下午教了一样新东西——网络安全基础。

不是理论。是实战。

第一步：流量加密。

"之前我们的消息是明文传输的。"马农在白板上写。"任何人只要能抓到光纤上的数据包，就能看到所有消息内容。"

他写了一个简化的加密方案：

`java
// 对称加密——简化版XOR加密
// 注意：这不是安全的加密算法。只是教学示范。
// 实际部署用AES-256。
public class SimpleEncrypt {
    
    private final byte[] key;
    
    public SimpleEncrypt(String keyStr) {
        // 用SHA-256把任意长度的密钥扩展为32字节
        this.key = sha256(keyStr.getBytes());
    }
    
    public byte[] encrypt(byte[] data) {
        byte[] result = new byte[data.length];
        for (int i = 0; i < data.length; i++) {
            result[i] = (byte) (data[i] ^ key[i % key.length]);
        }
        return result;
    }
    
    // 对称加密：加密和解密是同一个操作
    public byte[] decrypt(byte[] encrypted) {
        return encrypt(encrypted);  // XOR的逆运算就是XOR本身
    }
}
`

"XOR加密是最简单的对称加密。"马农说。"同一个密钥，加密一次得到密文，再加密一次就回到原文。因为任何数异或自己两次等于原值。"

"这安全吗？"赵天问。

"不安全。有明文攻击的漏洞。"马农说。"但它教你一个核心概念——对称加密的本质就是'用密钥把数据搅乱'。真正的AES-256做的事情更复杂，但原理相通。"

他在白板上画了通讯协议的升级：

`
通讯协议v2.0：
v1.0: [帧头][长度][明文数据][校验和][帧尾]
v2.0: [帧头][长度][加密数据][校验和][帧尾]
                    ↑
              用共享密钥加密
`

"广州端和我们共享同一个密钥。密钥通过串口的SDH备用通道发送——因为那条线路是物理隔离的，比光纤以太网更难被监听。"

第二步：接入控制。

`java
// 简易IP白名单
public class AccessControl {
    
    private final Set<String> allowedIPs = new HashSet<>();
    
    public void allow(String ip) {
        allowedIPs.add(ip);
    }
    
    public boolean check(String clientIP) {
        return allowedIPs.contains(clientIP);
    }
}
`

"白名单。只有我们认可的IP才能接入。"马农说。"简单但有效。"

第三步：日志审计。

"每一次接入、每一条消息、每一次连接失败——全部记录。"马农说。"日志是安全的眼睛。你看不到攻击者，但日志看得到。"

他给了林夕一个任务：把之前写的InventoryLog改造成通用的审计日志模块。

那天晚上。马农一个人在终端前。

他在做一件学员们不知道的事——追踪那条"友善提醒"的来源。

`bash
tcpdump -i eth0 src 10.200.3.0/24
`

抓包。分析。

10.200.3.17的数据包有一个特征——TTL值是56。标准TTL初始值通常是64（Linux）或128（Windows）。56意味着数据包经过了8跳路由才到达903室。

8跳。从10.200网段。

马农打开了他备份的BGP路由表——第10章从核心网络区拿到的那份1.7MB的文件。

他在路由表里搜索10.200段的路由：

`
10.200.0.0/16  via 100.64.2.1  AS 64512  origin: Tencent-IDC-DG (东莞)
`

东莞。

"友善提醒"来自东莞。和赵刚报告的"松山湖团队"是同一拨人。

马农把这个信息存了下来。没有告诉任何人。

他在教学笔记里加了一行：

`
CHEP后第34天。
外部势力开始出现。东莞。可能还有更多。
903的网络需要从"能用"升级到"安全"。
教学计划需要调整：第四周加入安全模块。
技术不是中性的。在资源匮乏的世界里，技术就是武器。
我不希望它变成武器。但我必须教学员们如何防御。
``

他关了灯。

窗外。深圳的第三十四个夜晚。

不知道从什么时候开始，903室的灯光变成了这片区域的一个标志——一个"有人在"的信号。

有人把这个信号当作希望。

有人把它当作目标。

马农知道。这只是开始。
