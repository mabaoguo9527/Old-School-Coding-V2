# Contract: 单章格式规范

**Version**: 1.0 | **Date**: 2026-03-09

---

## 文件规范

- **文件名**: `ch{NNN}.md`（三位数字零填充，如 ch001.md、ch050.md、ch300.md）
- **存放路径**: `novel/vol{N}/ch{NNN}.md`
- **编码**: UTF-8

## 章节结构模板

```markdown
# 第{N}章 {标题}

> {本章一句话概要，20字以内}

---

{正文内容，约3000字}

---

<!-- 
章节元数据（不输出到读者端）
- 字数: {实际字数}
- 技术点: {本章核心技术概念，1-2个}
- 武侠映射: {技术→武侠的核心比喻}
- 伏笔/回收: {埋下的伏笔 或 回收了哪个伏笔}
- 情感节拍: {重生反击/预言成真/技术封神/过渡/世界建设}
-->
```

## 字数规范

| 指标 | 要求 |
|------|------|
| 目标字数 | 3000字 |
| 允许波动 | 2500-3500字 |
| 硬底线 | ≥2000字 |
| 硬上限 | ≤4000字 |

## 段落配比

| 类型 | 占比 | 约字数 | 说明 |
|------|------|--------|------|
| 技术描写 | 10% | ~300字 | 武侠化技术叙事为主，代码≤2%（约60字），不写大段代码 |
| 叙事推进 | 50% | ~1500字 | 剧情发展、场景切换、行动描写（**核心主体**） |
| 情感/对话 | 40% | ~1200字 | 角色对话、内心活动、人物关系、情感表达 |

## 代码片段规范（⚠️ 全局约束：≤2%）

- **代码总量占比 ≤ 2%**（3000字章节中最多约60字/3-4行代码）
- 优先用**叙事性描写+武侠比喻**传递技术爽感，而非展示代码本身
- 代码仅作**点睛之笔**：一行关键代码胜过一整段代码块
- 单段代码长度：**最多3-4行**
- 禁止出现大段代码块（>5行）
- 代码语言：以Java为主，C/汇编为辅
- 代码前后 MUST 有叙事过渡（不能裸贴代码）
- 代码 MUST 技术准确
- **重点优先级：剧情推进 > 叙事张力 > 情感共鸣 > 技术细节**

## 章末钩子

每章最后1-3句 MUST 设置悬念或引导读者继续阅读：
- **技术钩子**: 揭示更深层的技术隐患（"但他没注意到，内核日志最后一行……"）
- **剧情钩子**: 新人物/新危机/新线索出现
- **情感钩子**: 角色关系的微妙变化或内心冲突加深
