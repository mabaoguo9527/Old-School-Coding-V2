# Contract: 技术描写规范

**Version**: 1.0 | **Date**: 2026-03-09

---

## 三层叙事结构

每段技术描写 MUST 同时满足三个层次的读者：

### Layer 1: 表层（所有读者）
- 用武侠比喻传递紧张感和史诗感
- 即使跳过技术细节也能理解"发生了什么、为什么危险、结果如何"
- 示例：*"堆区暴涨如真气逆行，经脉（线程栈）一条条断裂——走火入魔。"*

### Layer 2: 中层（有编程基础的读者）
- 术语准确，逻辑链完整
- 能感受到"行家出手"的内行爽感
- 示例：*"G1 GC的Mixed GC阶段在Old Region的回收速度追不上分配速度，触发了Full GC——Stop-The-World长达10秒。"*

### Layer 3: 深层（Java/系统专家）
- 隐藏技术彩蛋：代码片段暗合经典设计模式、调优参数有现实案例出处
- 示例：*代码中使用了 `@Contended` 注解消除伪共享——这是JDK 8+ `jdk.internal.misc.Contended` 的真实用法*

## 术语引入规则

| 规则 | 要求 |
|------|------|
| 首次出现 | MUST 附带一句话"武侠翻译"（如"CAS锁——以巧取胜的无锁之锁"） |
| 后续使用 | 可直接使用术语，无需重复解释 |
| 全书首次表 | 维护在 data-model.md 的术语表中 |
| 单章新术语上限 | ≤3个首次出现的新术语 |

## 技术→武侠映射表（核心）

| 技术概念 | 武侠映射 | 适用场景 |
|----------|---------|---------|
| 内存溢出（OOM） | 走火入魔、真气逆行 | 系统崩溃场景 |

| 高并发 | 千军万马过独木桥 | 流量洪峰场景 |
| CAS无锁 | 以巧取胜、得手即走 | 并发优化场景 |
| synchronized | 铁索连环阵——安全但笨重 | 对比教学场景 |
| 线程dump | 号脉 | Debug排查场景 |
| 日志分析 | 望诊 | 故障排查场景 |
| 内存快照 | 开膛验尸 | 深度排查场景 |
| ClassLoader双亲委派 | 军队指挥链/层层上报 | 教学场景 |
| 代码重构 | 易筋经/洗髓经 | 架构升级场景 |
| 微服务拆分 | 分兵把守各隘口 | 架构设计场景 |
| 消息队列 | 粮草辎重线 | 流量削峰场景 |
| 负载均衡 | 中军调度 | 分布式场景 |
| 熔断器 | 不让一个士兵溃败拖垮整条防线 | 容错场景 |
| 火焰图 | 透视眼/内视 | 性能分析场景 |
| 分布式事务 | 千里传书、信使往返 | 数据一致性场景 |
| 内存屏障 | 结界/封印 | 并发可见性场景 |
| 字节码 | 内功心法的底层口诀 | JVM深度场景 |

## 技术准确性要求

| 项目 | 标准 |
|------|------|
| Java版本 | JDK 17 LTS（API、类名、方法签名 MUST 真实存在） |
| JVM参数 | MUST 是HotSpot VM真实支持的参数 |
| 代码片段 | 核心逻辑 MUST 可编译通过（允许省略import） |
| Spring Boot | 基于2.x/3.x真实自动装配原理 |
| 网络协议 | TCP/IP四层模型准确，端口号/协议名真实 |
| ARM汇编 | 指令集 MUST 属于真实ARM架构 |

## ⚠️ 全局约束：代码量 ≤ 2%

| 项目 | 要求 |
|------|------|
| 每章代码占比 | **≤ 2%**（3000字章节中最多约60字/3-4行代码） |
| 代码用途 | 仅作为**点睛之笔**，一行关键代码胜过一整段代码块 |
| 禁止事项 | 不得出现大段代码块（>5行）、不得用代码推动叙事 |
| 替代手段 | 用**叙事描写+武侠比喻+术语口语化**传递技术爽感 |
| 重点优先级 | **剧情推进 > 叙事张力 > 情感共鸣 > 技术细节** |

### 代码展示方式（优先级从高到低）
1. **叙述嵌入**：将代码动作融入叙事（"他敲下 `volatile` 关键字，如同在记忆的裂缝上打下一枚铁钉"）
2. **单行引用**：行内用反引号引用关键代码片段（`System.gc()`）
3. **极简代码块**：最多3-4行，仅展示最关键的逻辑转折点
4. **伪代码/注释**：用中文注释替代完整代码，传递思路而非实现

## 技术描写节奏模板

```
【铺垫】叙事性描写设置紧张感（2-3句）
    ↓
【症状】用武侠比喻描述技术问题的表象（1-2句）
    ↓
【诊断】马农的分析过程，穿插术语（3-5句）
    ↓
【关键一招】叙述性地描写技术操作，可嵌入1行关键代码（2-3句）
    ↓
【解读】将技术动作翻译为武侠场景（2-3句）
    ↓
【结果】技术操作的效果+旁人反应+情感收束（2-3句）
```
