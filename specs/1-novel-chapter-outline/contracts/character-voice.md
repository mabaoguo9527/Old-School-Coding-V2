# Contract: 角色语言风格规范

**Version**: 1.0 | **Date**: 2026-03-09

---

## 马农（主角）

### 对话风格
- **字数**: 单句≤20字为主，极少超过30字
- **语气**: 平淡如水，永远不急不躁
- **特征**: 技术术语自然嵌入，不解释除非在教学场景
- **禁忌**: MUST NOT 长篇大论、MUST NOT 感叹号过多、MUST NOT 说教式台词

### 示例库

| 场景 | 台词示例 |
|------|---------|
| 全场慌乱 | "堆区还有200MB。够了。" |
| 被质疑 | "跑一遍就知道了。" |
| 教学 | "双亲委派。你不会的事交给上级，上级不会再往上。" |
| 拒绝 | "代码是用来解放人的，不是控制人的。" |
| 评价代码 | "变量名。重新起。" |
| 认可 | "可以。"（最高评价） |
| 情感流露 | "师父说，代码的尽头是人。"（罕见，仅关键节点） |
| 预言 | "48小时后，城市回到1990年。" |

### 内心独白
- 极少直接写内心活动
- 用行为暗示心理：如"他看着屏幕上的代码，手指停了三秒，然后继续敲下去"
- 罕见的内心独白 MUST 简短，如"不够。还不够。"

## 林夕（第一门徒）

### 对话风格
- **字数**: 正常句长，20-40字
- **语气**: 早期怯弱好奇，中期自信果断，后期沉稳专业
- **特征**: 会问"为什么"，技术术语从生涩到熟练的成长线
- **成长标志**: 从问马农"这是什么"到自己解释给新人

### 示例库

| 阶段 | 台词示例 |
|------|---------|
| 早期（卷一） | "这个……for循环，是让它重复做一件事？" |
| 中期（卷二） | "分布式锁不能用synchronized，需要跨节点共识。我来写。" |
| 后期（卷三） | "马农教过我——熔断器不是认输，是保存实力。" |
| 成熟（卷四） | "各位，这段代码的设计思路是……"（独立技术讲解） |

## 沈墨（天才叛徒）

### 对话风格
- **字数**: 简短锐利，偏短句
- **语气**: 早期傲慢急躁，叛逃时冷漠，回归后沉稳
- **特征**: 天才式的直觉性表达，"感觉"比"推理"多
- **转折标志**: 从"我觉得能跑就行"到"让我再检查一遍"

### 示例库

| 阶段 | 台词示例 |
|------|---------|
| 入门（卷二） | "这段代码能跑就行，为什么要重写？" |
| 冲突 | "你的规矩太多了。代码是自由的。" |
| 叛逃后 | "周崖说得对，速度比规矩重要。" |
| 回归 | "我错了。能跑的代码和能活的代码是两回事。" |
| 成熟（卷四） | "这段代码的签名校验有异常，我需要逐行审计。" |

## 周崖（师兄/对手）

### 对话风格
- **字数**: 中长句，善于煽动
- **语气**: 魅力型领袖，有说服力，但底层有控制欲
- **特征**: 用技术类比阐述权力哲学
- **与马农对比**: 马农用代码说话，周崖用话术包装代码

### 示例库

| 场景 | 台词示例 |
|------|---------|
| 招揽 | "师弟，你我联手，天下代码尽归我们写。" |
| 对立 | "你太天真了。人不配拥有自由的代码。" |
| 威胁 | "72小时。交出骨干网控制权，或者全面战争。" |
| 决斗 | "师父的内存屏障锁定术……你也会。" |
| 投降 | "我输了。不是输给你的代码，是输给师父那句话。" |

## 陈博士

### 对话风格
- **字数**: 偏长，学术腔
- **语气**: 温和理性，但有时天真
- **特征**: 喜欢用学术概念解释实践问题
- **转折**: 从"理论上可行"到"我错了，实践才是检验"

### 示例库

| 场景 | 台词示例 |
|------|---------|
| 早期 | "从理论上说，受控重启Origin的子模块是最优路径。" |
| 被揭露 | "不可能……我的系统怎么会有后门？我亲自审查过的。" |
| 觉醒 | "马农，你说得对。我太相信'优雅的解决方案'了。" |
| 合作后 | "逆向分析显示，注解处理器在编译期注入了隐藏线程。" |

## 叙述者（第三人称限知视角）

### 叙事语言规范
- **技术场景**: 短句、动词密集、节奏如武侠过招
  - ✅ "指尖落下。`volatile`关键字锁定可见性。内存屏障立起。三个线程同时读到最新值。"
  - ❌ "他使用了volatile关键字来确保内存可见性，这样三个线程就都能看到最新的值了。"
- **情感场景**: 克制，通过技术隐喻传递
  - ✅ "他看着满屏的legacy code，像看一个老朋友的遗书。"
  - ❌ "他非常伤感地看着屏幕上的旧代码，心里五味杂陈。"
- **全文MUST NOT**:
  - 网文套路打脸桥段
  - 后宫元素
  - 无脑升级系统
  - 过度煽情/卖惨
  - "震惊""竟然""居然"等廉价惊叹词的滥用
